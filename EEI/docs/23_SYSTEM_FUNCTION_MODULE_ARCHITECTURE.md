# 23 - 系统功能板块、导航与架构体系

## 结论

产品是由统一数据快照、模型版本和分析上下文驱动的商业情报可视化工作台。正式导航包含 **17 个板块**；功能目录是 `data/function_catalog.csv`。

## 导航结构

```text
探索
  商业版图 / 全链供应链 / 集团业务与子板块 / 资金与并购
  所有权与控制 / 政策与政府 / 战略信号 / 行业地图
研究管理
  Watchlist / 重要变化 / 已保存视图
数据与模型
  数据库与来源 / 模型与参数 / 操作日志 / 双周校准
系统治理
  功能结构 / 开发治理
```

## 页面骨架

左侧导航与 Watchlist；顶部主体/搜索/快照/模型；中央主可视化；右侧对象/证据/贡献；底部时间、数据和刷新状态。首页可视化覆盖 >=90，核心页面平均 >=80。

## 共享上下文

`focus_entity_id`、`as_of`、`data_snapshot_id`、`score_snapshot_id`、`profile_version`、关系/地区/证据筛选、探索路径、选中对象、视口和 motion preference。切换板块不丢失状态。

## 技术层

Web UI -> 图/Sankey/时间线/矩阵/ERD -> API 投影与模型预览 -> 领域服务 -> PostgreSQL、原始快照、证据、配置、分数快照、outbox 与日志。

完整功能矩阵：`FUNCTION_CATALOG.md`。
