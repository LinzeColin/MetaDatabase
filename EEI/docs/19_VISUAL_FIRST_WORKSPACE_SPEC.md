# 19 - 可视化优先研究工作台规格

## 1. 产品命题

首页不是“进入研究的导航页”，首页就是“正在进行的研究”。主图是第一信息层，文字是解释层，表格是核验与无障碍层。

## 2. 默认状态决策树

```text
打开产品
  ├─ 有最近会话 -> 恢复最近主体 + 视角 + 时间 + 路径
  ├─ 无最近会话但有 Watchlist -> 打开未读重要变化最高的对象
  └─ Watchlist 为空 -> 主画布显示行业入口 + NVIDIA 演示
```

不得先跳转到独立行业卡片页。行业入口应在左侧上下文栏和画布空状态中出现。

## 3. 页面骨架

```text
┌────┬────────────┬──────────────────────────────────┬──────────────┐
│主导│Watchlist /  │ 顶部：搜索、探索路径、截至日期     │ 所选对象详情  │
│航栏│行业 / 最近  ├──────────────────────────────────┤              │
│    │            │                                  │ 摘要         │
│    │            │      当前公司的可视化商业版图      │ 关键连接     │
│    │            │                                  │ 证据状态     │
│    │            ├──────────────────────────────────┤ 操作         │
│    │            │ 时间变化 / 历史快照                │              │
└────┴────────────┴──────────────────────────────────┴──────────────┘
```

桌面端推荐宽度：导航 64-68px，上下文栏 240-280px，详情栏 320-380px，主画布使用剩余空间。低于 1150px 时详情栏变为可唤出的抽屉；低于 820px 时上下文栏也折叠。

## 4. 主图布局

### 4.1 方向约定

- 上游在左：原材料、设备、IP/EDA、制造、封装、能源。
- 中：当前主体及其业务板块、产品组、子公司和设施。
- 右：系统商、云平台、渠道、行业客户和最终需求。
- 上：所有权、治理、融资、并购和资本关系。
- 下：政策、监管、出口限制、补贴、政府合同和外部风险。

### 4.2 关系视觉语法

| 维度 | 编码 |
|---|---|
| 方向 | 箭头 + 起终点语言标签 |
| 关系族 | 颜色 + 图标 + 标签；颜色不单独承载含义 |
| 证据状态 | 实线=公开披露；虚线=多源推断；双线/警告=信息冲突 |
| 重要性 | 线宽与排序；用户可关闭 |
| 时间 | 时间条、淡入淡出和明确日期标签 |
| 聚合 | 带数量的组节点，例如“设备供应商 18 家” |
| 可探索性 | 节点边框和 hover/focus 状态；右侧明确“以它为中心” |

### 4.3 反 hairball 规则

- 首屏最多 40 条边；推荐 20-30 条。
- 单次展开默认 10 个对象；需要继续展开时展示数量和排序依据。
- 同一类型超过 6 个节点时优先聚合。
- 节点重叠、标签遮挡或边交叉达到阈值时强制重新布局/聚合。
- 用户切换视角时保持主节点和已固定节点的空间位置，减少认知跳变。

## 5. 分析视角

默认“综合”。其他视角不是新页面，而是对同一工作台的关系层过滤：

1. 供应链：材料、设备、制造、封装、系统、能源、物流、客户。
2. 资金与交易：融资、投资、并购、回购、资本开支、重大合同。
3. 股权与控制：母子公司、持股、投票权、董事与控制安排。
4. 业务板块：集团、分部、品牌、产品、设施、市场。
5. 政策与风险：监管、补贴、政府合同、出口限制、诉讼和地缘风险。
6. 战略变化：招聘、产能、产品、合作、供应商迁移和重大事件。

## 6. 状态持久化

保存视图至少记录：当前主体、所选节点、分析视角、时间快照、关系过滤、语义缩放级别、固定节点、比较对象、画布位置、打开面板、用户备注和探索路径。URL 用于可分享状态；数据库用于长期保存；本地缓存用于未登录恢复。
