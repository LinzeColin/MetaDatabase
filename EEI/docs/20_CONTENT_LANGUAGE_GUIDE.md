# 20 - 人类语言与内容设计指南

## 1. 原则

- 先说对象和关系，再说模型与字段。
- 用动词表达关系，例如“为 NVIDIA 提供晶圆制造”，而不是只显示 `FOUNDRY_SUPPLIER_OF`。
- 结论、证据和不确定性分开。
- 默认显示可理解的等级；数值、公式和归一化方法放在“查看计算方法”。
- 不把研究假设写成事实，不把重要性评分写成收益概率。

## 3. 标题模板

- 页面：`{公司名} 商业版图`
- 区块：`关键上游`、`主要客户与去向`、`业务板块`、`资金与交易`、`政策与风险`
- 关系：`{起点} 为 {终点} 提供 {产品/能力}`
- 变化：`{日期} · {主体} 新增/终止/扩大/缩减 {关系或事件}`
- 数据状态：`截至 {日期}`、`资料完整度 {等级}`、`更新延迟 {范围}`

## 4. 不确定性模板

- 公开披露：`该关系可由一手公开文件直接核验。`
- 多源推断：`多个公开来源支持该关系，但未发现单一完整披露。`
- 信息冲突：`不同来源对该关系存在不一致，需要人工复核。`
- 尚未披露：`当前公开资料不足，不能判定为不存在。`
- 数据滞后：`该信息反映报告期状态，不代表当前实时资金流。`

## 5. 操作文案

主动作使用动词：`以它为中心`、`展开上游`、`展开下游`、`查看路径`、`加入关注`、`保存当前视图`、`查看来源`。避免“执行”“提交任务”“运行流程”等 Agent 化语言。
