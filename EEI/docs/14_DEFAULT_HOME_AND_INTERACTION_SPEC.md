# 14 - 默认首页与交互规格（v4.2）

## 1. 已批准默认视图

默认首页即“可视化研究台”，不再使用行业卡片页。左侧展示 Watchlist 和行业入口；中央立即打开一家公司商业版图；右侧展示所选对象详情；底部展示时间变化。

## 2. 主体选择优先级

1. 恢复上次会话。
2. 打开 Watchlist 中未读重要变化最高的公司。
3. Watchlist 为空时，在画布内提供行业选择和 NVIDIA 演示。

## 3. 首屏必须出现

- 当前公司名称和截至日期。
- 可视化上下游结构。
- 至少一个明确可点击的上游和下游节点。
- 分析视角切换。
- Watchlist。
- “以它为中心”操作。
- 资料完整度、更新延迟和证据状态入口。

## 4. 左侧上下文栏

`关注 / 行业 / 最近` 三个页签。Watchlist 行显示名称、行业、未读变化和数据状态；点击后直接更新中央图。行业行打开行业局部图或代表对象，不跳转到卡片仪表盘。

## 5. 中央主画布

使用稳定的左右方向：上游左、主体中、下游右。业务板块围绕主体；资本与控制上置；政策与风险下置。默认只显示最重要关系，其余按组折叠。

## 6. 右侧对象详情

单击节点后展示自然语言摘要、与当前公司的关系、三条关键连接、资料状态和主动作。默认主动作是“以它为中心”，次动作是“展开上游 / 展开下游 / 加入关注 / 查看路径”。

## 7. 持久化

保存当前主体、所选节点、分析视角、时间、过滤、语义缩放、固定节点、比较对象、画布状态、备注和 breadcrumb。浏览器刷新后恢复；保存视图可长期恢复和分享。

## 8. P0 验收

- 有 Watchlist 时打开产品不出现独立行业卡片页。
- 主图占首屏可用面积至少 60%。
- 用户两次以内操作完成“把 TSMC 设为中心”。
- 连续切换三次后 breadcrumb、返回、URL 和状态继承正确。
- 关系逻辑优先以视觉呈现，文字仅解释与核验。
- 列表模式与图谱显示同一关系集合。
- 主 UI 通过禁用词静态检查。
