# 36 - Release Gates 与 Definition of Done

## DoD

功能只有在规格、实现、测试、视觉、数据语义、可访问性、风险、回滚和文档全部完成后才可标记 DONE。原型完成不等于生产完成。

## G0-G9

| Gate | 核心目标 |
|---|---|
| G0 | 计划、治理、范围、风险、验收和回滚 |
| G1 | 可复现仓库与基础设施 |
| G2 | 领域模型、迁移和目录 |
| G3 | 入口、Watchlist、行业与配置目录 |
| G4 | 递归探索、共享上下文、数据库和治理工作台 |
| G5 | 八层公司视图、交互、动画和视觉覆盖 |
| G6 | 模型、解释、日志与双周校准 |
| G7 | 数据接入、质量、新鲜度和变更检测 |
| G8 | 资本、政策、战略和跨视图一致性 |
| G9 | 性能、安全、可访问、回滚、文档和发布 |

机器目录：`data/release_gate_catalog.csv`。
