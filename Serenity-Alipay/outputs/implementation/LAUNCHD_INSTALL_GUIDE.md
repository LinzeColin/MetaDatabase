## Install Or Refresh

```bash
mkdir -p ~/Library/LaunchAgents
cp outputs/implementation/com.serenity.daily-analysis.plist ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
plutil -lint ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
launchctl enable "gui/$(id -u)/com.serenity.daily-analysis"
```

Kick once for smoke:

```bash
launchctl kickstart -k "gui/$(id -u)/com.serenity.daily-analysis"
```

Check logs:

```bash
tail -n 80 data/serenity_launchd.out.log
tail -n 80 data/serenity_launchd.err.log
```

Current loaded-state check:

```bash
launchctl list | rg 'com\.serenity\.daily-analysis'
```

## Production Runtime Gate

Runtime production execution evidence should use:

```bash
SERENITY_MAIL_SEND_ENABLED=true python -m app.cli preflight --require-production --json
```

The installed launchd plist remains shadow-safe unless you explicitly replace it with the production-mail plist. Codex app cron automations are active separately and run the preflight-gated production tick command.

After production data gates pass, enable real email only for a controlled smoke:

```bash
SERENITY_MAIL_SEND_ENABLED=true python -m app.cli mail-smoke --send --confirm-real-send SEND --require-send-ready --json
```

Generate the production-mail launchd template and checklist:

```bash
python -m app.cli mail-unlock-check --json
```

Only after the real-send smoke succeeds should launchd use real send:

```bash
cp outputs/implementation/com.serenity.daily-analysis.production-mail.plist ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
plutil -lint ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
launchctl kickstart -k "gui/$(id -u)/com.serenity.daily-analysis"
```

Rollback to shadow-mail config:

```bash
cp outputs/implementation/com.serenity.daily-analysis.plist ~/Library/LaunchAgents/com.serenity.daily-analysis.plist
launchctl kickstart -k "gui/$(id -u)/com.serenity.daily-analysis"
```
