# 05 UI Pages

MVP is CLI + Markdown report, not a full web app. A web dashboard can be V2.

## MVP Report Sections

1. Run Status
2. Top5 Candidate Pool
3. Target Weight Recommendation
4. Current Holdings Audit
5. Rebalance Trigger Matrix
6. Benchmark Comparison
7. Drawdown And Recovery-Time Check
8. Fees And Execution Rules
9. Evidence Chain
10. Missing Data / Conflicts
11. Notification Preview
12. Next Action Deadline

## Optional V2 Dashboard Pages

| Page | Purpose |
|---|---|
| Run History | Daily run list and status |
| Top5 Timeline | Top5 changes across day/week/month |
| Holding Discipline | Current vs target deviation |
| Candidate Detail | Evidence, fees, drawdown, benchmark data |
| Manual Review Queue | Items requiring user confirmation |
| Data Quality | Missing, stale, conflicting sources |
| Notification Log | Sent/draft/local notification records |

## UX Requirements

- Dense and operational, not marketing-style.
- Prioritize scanability.
- Every action label must show reason and source freshness.
- Risk gates must be visibly separate from score.
- No hidden downgrade.
