使用说明

说明 Skill 选股逻辑、策略口径、权重配置公式和持仓调整纪律；本说明不构成自动交易指令。

Skill 选股逻辑

Serenity 不是先拿一张规则表机械筛选，而是先建立“未来 1-3 个月最可能获得资金和业绩弹性的高成长方向”假设，再用证据验证这个假设能否落到可执行的场外基金上。

  • 如何挑选：先看高成长主题是否仍有景气度、资金关注和可解释的上涨来源，再把主题映射到能在场外买到的基金，而不是直接从基金列表里随便排序。
  • 怎么挑选：先按产业链卡点、稀缺层、主题暴露和 1-3 个月资金弹性形成 Serenity 优先级，再映射到能在场外买到的基金。
  • 为什么挑选：Top5 先由 Serenity 判断决定，不由 Score 机械排序；如果一只基金分数高但不在更关键的产业链位置，它不能越过更高 Serenity 优先级。
  • 为什么调仓：当新一轮证据显示产业链强弱、资金方向、风险回撤、费用/申赎状态或 Top5 排名发生变化，系统会把它解释为原 baseline 假设需要修正，并输出增配、减少、暂停新增或维持。
  • 思考公示方式：页面展示证据置信度、等级、目标权重、基准权重、相对比例、费用状态、基金库证据和运行时间线，让用户能看到“为什么是这只、为什么是这个权重、为什么现在动或不动”。

证据置信度

ConfidenceScore = Data 25 + Timeliness 15 + Source 15 + Return 15 + Risk 20 + Executable 10

  • 证据置信度不是选股主排序，只回答“Serenity 判断有多少数据支持、能不能执行”。
  • Data = max(0, 25 - 8 x 缺失字段数)。
  • Timeliness = 净值/持仓缺失不超过 2 天得 15，否则 0。
  • Source = 官方级来源至少 2 个且无冲突得 15；至少 1 个得 7.5；否则 0。
  • Return = 15 x 跑赢次数 / 6，比较窗口为 1 个月、3 个月、10 交易日，对照沪指和标普 500。
  • Risk = MDD 小于 40.00% 才计分；回撤修复时间达到 365 天会强制压低风险分。
  • Executable = 申购/赎回开放且费率分档完整才得 10，否则 0。

等级与策略口径

  • ConfidenceScore >= 85：Action-Ready，说明 Serenity 判断的证据和执行条件足够强。
  • 70-84：Watch，Serenity 判断可保留，但证据或执行条件需要继续观察。
  • 55-69：Manual Review，必须人工复核后处理。
  • <55：Block/skip，不执行调仓建议。
  • MDD >= 40.00%、保守类资产、来源冲突、关键费率缺失会触发硬降级。

权重配置公式

SerenityRank_i = 产业链卡点优先级 + 主题暴露 + 场外可执行性

ConfidenceModifier_i = 0.85 + 0.15 x ConfidenceScore_i / 100

RawWeight_i = SerenityBase_i x ConfidenceModifier_i

Capped_i = min(normalize(RawWeight_i), 30.00%)

TargetWeight_i = Capped_i / sum(Capped)

  • 仅非 Block 候选参与 Top5；排序优先服从 Serenity 判断，证据置信度只做辅助修正。
  • 低 Serenity 优先级标的不会仅凭更高 ConfidenceScore 超过高优先级标的。
  • 目标权重是策略份额，不是支付宝真实账户仓位。
  • 首轮基准从 0.00% 开始；后续运行相对上一轮 Serenity baseline 比较。

持仓调整逻辑

这里不是把 Deviation 翻译成买卖术语，而是说明系统如何从 Serenity 新判断推导到人工操作建议。

  • Deviation = TargetWeight - BaselineWeight。
  • 凭什么：先确认数据质量通过、基金申赎和费率可执行、候选没有 Block，再比较新一轮 Serenity 目标权重和上一轮 Serenity baseline。
  • 为什么：偏离不是账户盈亏，而是 Serenity 对产业链优先级、主题弹性、风险和执行条件的最新判断相对旧 baseline 的变化。
  • 怎么做：|Deviation| <= 1.00%：维持；Deviation > 1.00% 且 Action-Ready：增配；非 Action-Ready：暂停新增或人工复核；Deviation < -1.00%：减少；Block：阻断或清仓标签。
  • 做多少：策略调整份额 = TargetWeight - BaselineWeight；若需要换算金额，人工按“计划投入资金 x |Deviation|”在支付宝或官方平台确认，系统不自动下单。
  • 为什么做这么多：TargetWeight 已由 Serenity 优先级、证据置信度修正、30.00% 单标上限和归一化约束计算完成；Deviation 是把旧 baseline 调到新目标所需的最小策略差额。
  • 为什么 1.00% 内维持：小于等于 1.00% 的变化通常不足以覆盖确认成本、申赎费、净值时差和短时噪声，除非同时触发风险硬门槛。

重平衡触发

  • 单标偏离超过 1.00% 会触发纪律事件。
  • Top5 变动率超过 20.00%、新增 1 只、替换 2 只会触发复核。
  • 关键字段变化超过 1σ 会触发 regime check。
  • 同日、前一日、前一周、前一月都会生成对比快照。

人工复核与降级

  • 连续缺失净值/持仓超过 2 天，进入 Manual Review。
  • 申购费、赎回费、申购状态、赎回状态或费率分档缺失，暂停新增。
  • 官方级来源少于 2 个或来源冲突，不能 Action-Ready。
  • 执行锁开启时，只输出研究排序和纪律标签，不给新增订单。

如何使用

  • 先看“当前持仓建议”，确认最新时间和动作颜色。
  • 再看“持仓建议”表，切换初始持仓权重或上轮对比权重。
  • 点击“基金库”核对申赎状态和费用分档。
  • 若出现人工复核，先补证据或记录复核结论，再考虑平台操作。
  • 所有真实申购、赎回、增配、减配都必须在支付宝或官方平台人工确认。
