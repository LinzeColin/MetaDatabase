使用说明

用来解释 Serenity 为什么挑选、为什么调仓、怎么计算权重；Score 只表示证据置信度。

第一层

Skill 选股逻辑

Serenity 不是先拿一张规则表机械筛选，而是先形成“未来 1个月-1年最值得承担高波动的高成长方向”判断，再检查这个判断能否落到可执行的场外基金上。

如何挑选先看高成长主题是否仍有景气度、资金关注和可解释的上涨来源，再把主题映射到能在场外买到的基金。

怎么挑选按产业链卡点、稀缺层、主题暴露、资金弹性和执行可得性形成 Serenity 优先级。

为什么挑选Top5 先由 Serenity 判断决定，不由 Score 机械排序；低 Serenity 优先级标的不会仅凭更高 ConfidenceScore 超过高优先级标的。

为什么调仓当产业链强弱、资金方向、风险回撤、费用/申赎状态或 Top5 排名改变，系统会视为旧 baseline 需要修正。

页面公开证据置信度、等级、目标权重、基准权重、相对比例、费用状态、基金库证据和运行时间线，方便追溯每一次建议。
第二层

证据置信度

Score 不替代 Serenity 判断，只回答“这次判断有多少数据支持、是否足够可执行”。

ConfidenceScore = Data 25 + Timeliness 15 + Source 15 + Return 15 + Risk 20 + Executable 10

Data普通关键字段缺失按项扣分；return_windows 缺失按关键缺失处理，最高扣 20 分并触发复核。

Timeliness净值/持仓缺失不超过 2 天得分，否则进入时间完整性降级。

Source官方级来源至少 2 个且无冲突才完整得分；来源冲突优先解释冲突。

ReturnReturn = 15 x 跑赢次数 / 8；比较 1个月、3个月、1年、10交易日，对照沪指和标普500。

RiskMDD 小于 40.00% 才可继续；回撤修复时间达到 365 天会强制压低风险分。

Executable申购/赎回开放且费率分档完整才完整得分。

第三层

权重配置

SerenityRank_i = 产业链卡点优先级 + 主题暴露 + 场外可执行性
ConfidenceModifier_i = 0.85 + 0.15 x ConfidenceScore_i / 100
RawWeight_i = SerenityBase_i x ConfidenceModifier_i
Capped_i = min(normalize(RawWeight_i), 30.00%)
TargetWeight_i = Capped_i / sum(Capped)

排序仅非 Block 候选进入 Top5；排序优先服从 Serenity，ConfidenceScore 只做辅助修正。

权重目标权重是策略份额，不是支付宝真实账户仓位。

基准首轮基准从 0.00% 开始；后续运行相对上一轮 Serenity baseline 比较。

第四层

持仓调整逻辑

这里不是把 Deviation 翻译成买卖术语，而是说明系统如何从 Serenity 新判断推导到人工操作建议。

Deviation = TargetWeight - BaselineWeight

凭什么先确认数据质量通过、基金申赎和费率可执行、候选没有 Block，再比较新一轮 Serenity 目标权重和上一轮 Serenity baseline。

为什么偏离不是账户盈亏，而是 Serenity 对产业链优先级、主题弹性、风险和执行条件的最新判断相对旧 baseline 的变化。

怎么做|Deviation| <= 1.00%：维持；Deviation > 1.00% 且 Action-Ready：增配；非 Action-Ready：暂停新增或人工复核；Deviation < -1.00%：减少；Block：阻断或清仓标签。

做多少策略调整份额 = TargetWeight - BaselineWeight；若需要换算金额，人工按“计划投入资金 x |Deviation|”在支付宝或官方平台确认。

为什么做这么多TargetWeight 已由 Serenity 优先级、证据置信度修正、30.00% 单标上限和归一化约束计算完成；Deviation 是把旧 baseline 调到新目标所需的最小策略差额。

为什么 1.00% 内维持小于等于 1.00% 的变化通常不足以覆盖确认成本、申赎费、净值时差和短时噪声，除非同时触发风险硬门槛。

第五层

重平衡、复核与边界

重平衡单标偏离超过 1.00%、Top5 变动率超过 20.00%、新增 1 只、替换 2 只或关键字段变化超过 1σ，会触发纪律事件。

人工复核连续缺失净值/持仓超过 2 天、费率分档缺失、官方级来源少于 2 个或来源冲突，会进入 Manual Review。

为什么需要复核每个复核对象都会显示进入复核的具体原因，例如费率/申赎状态缺失、来源冲突、净值或持仓时间滞后、数据质量不足或执行条件不明确。
