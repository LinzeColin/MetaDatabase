# 功能清单

## 中文速读

Serenity-Alipay 是一个本地优先的基金研究和纪律审计系统。它每天在北京时间交易时段运行，基于场外基金候选、历史净值、基金规则、基准收益、数据质量和纪律规则，生成当前持仓建议、观察池、报告、首页数据和必要的邮件提醒。

它不会自动买入、卖出、申购或赎回。所有交易动作都必须由用户在支付宝基金、MooMoo 或对应官方平台人工确认。

当前 GitHub 状态：代码、测试、三基文件和开发记录均以 `LinzeColin/CodexProject/Serenity-Alipay` 为权威来源。历史 SQLite、报告、通知草稿属于运行态数据，不能直接混入源码主线。

## 核心功能

| 功能 ID | 中文名称 | 当前状态 | 功能说明 | 用户能看到什么 |
|---|---|---|---|---|
| `FEAT-SER-001` | 候选基金评分与风险门 | 已启用 | 检查基金候选的净值、持仓、费率、来源、收益、回撤、可执行性，并给出等级和动作标签。 | 每只基金的分数、等级、剔除原因、人工复核原因。 |
| `FEAT-SER-002` | Top5 排序与目标权重 | 已启用 | 先尊重 Serenity 候选优先级，再用分数作为置信度修正，筛出持仓池 Top5 并计算目标权重。 | 首页“当前持仓建议”和每只基金的目标比例。 |
| `FEAT-SER-003` | 对比与纪律告警 | 已启用 | 比较本轮和上轮、前一日、前一周、前一月的变化，发现 Top5 替换、权重偏离、单只过度放大等纪律风险。 | 调仓建议、观察池、风险提示、重平衡候选。 |
| `FEAT-SER-004` | 北京工作日自动运行安全 | 已启用 | 只在北京时间工作日 slot 内运行；非运行点只记录 tick；预检失败时降级，不硬下结论。 | `/api/scheduler/status`、运行日志、首页更新时间。 |
| `FEAT-SER-005` | 邮件频率控制 | 已启用 | 只有当前建议动作相对上一封已发送的 actionable 邮件发生实质变化时才发；北京时间同一天最多 2 封 actionable 邮件。 | 重要动作邮件；重复结论只写数据库和首页日志，不打扰用户。 |

## 用户最关心的输出

| 输出 | 说明 | 是否会自动交易 |
|---|---|---|
| 当前持仓建议 | 当前 Top5 和目标权重。 | 否 |
| 观察池 | 排名在持仓池之后、值得继续观察的候选。 | 否 |
| 需操作行为 | 增配、减配、暂停新增、清仓、人工复核等纪律标签。 | 否 |
| 正式报告 | 每次运行生成的中文分析报告。 | 否 |
| 邮件提醒 | 只有实质动作变化且未超过频率上限时发送。 | 否 |
| 数据库归档 | 保存运行、评分、推荐、复核、通知日志，支持后续对比。 | 否 |

## 明确不做的事

| 不做事项 | 原因 |
|---|---|
| 自动下单 | 用户必须在平台人工确认申购或赎回。 |
| 因支付宝暂不支持就直接剔除基金 | 平台交易可用性只做建议，不作为硬剔除条件。 |
| 缺失数据时硬下结论 | 数据不足时必须降级为观察、人工复核或暂停新增。 |
| 覆盖历史快照和旧分析 | 历史是当时事实，不能用当前认知回写污染。 |
| 把运行态 SQLite/WAL/lock 提交到 main | 这些是本机状态，不属于源码主线。 |
