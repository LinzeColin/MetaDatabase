# 27 - 丝滑动画、交互反馈与可选触觉系统

## 1. 结论

动画不是装饰，而是用于解释“对象从哪里来、去了哪里、什么发生变化”。系统使用统一 motion tokens，并提供视觉、文字、键盘焦点和可选触觉四类反馈。触觉只在支持的移动设备和用户开启时使用，桌面端不得依赖它。

## 2. 反馈层级

| 用户动作 | 即时视觉 | 空间/数据反馈 | 文字反馈 | 可选触觉 |
|---|---|---|---|---|
| Hover/Focus 节点 | 40-80ms 高亮、连接线增强 | 邻接节点轻微显现 | tooltip | 无 |
| 选择节点 | 选择环 + 12px 脉冲 | 详情抽屉 180-240ms 进入 | ARIA live 简述 | 10-15ms |
| 设为中心 | 节点向中心移动 | 旧邻域淡出、新邻域 morph，320-420ms | breadcrumb 更新 | 15-20ms |
| 切换镜头 | 当前层减弱 | 图形稳定对象保留位置，220-360ms | 视角标签 | 10ms |
| 修改参数 | 滑块和数值同步 | 节点大小、边宽、排序连续过渡 | “预览已更新” | 8-12ms，节流 |
| 保存/激活 | 按钮成功态 | snapshot 状态变化 | toast + 日志 id | 20ms |
| 错误 | 边框/图标/文本三重提示 | 不进行大幅抖动 | 明确错误与恢复 | 30-40ms，单次 |

## 3. Motion tokens

```json
{
  "instant": 80,
  "local": 160,
  "panel": 220,
  "data_update": 280,
  "lens_change": 320,
  "reroot": 380,
  "full_relayout_max": 480
}
```

默认 easing：

- 局部反馈：`cubic-bezier(0.2, 0, 0, 1)`；
- 进入：`cubic-bezier(0, 0, 0, 1)`；
- 离开：`cubic-bezier(0.3, 0, 1, 1)`；
- 图谱空间变化：弹性极低的 spring 或等价 bezier，禁止明显回弹。

## 4. 图谱动画规则

1. 对象 id 稳定，数据更新使用 morph，不销毁整张图；
2. 主体切换时选中节点成为锚点；
3. 未变化节点尽量保持位置；
4. 新节点从其父/关联节点附近出现；
5. 删除节点淡出并收拢；
6. 边先更新方向和状态，再更新标签；
7. 大于 120 个动画对象时降级为分批或淡入；
8. 动画期间仍允许取消/返回，不锁死 UI；
9. 所有动画可通过 `prefers-reduced-motion` 或应用设置降低。

## 5. 多模态边界

- 视觉：颜色 + 形状 + 线型 + 大小，不用颜色单独表达；
- 触觉：使用 `navigator.vibrate` 的渐进增强，默认关闭，可用时才显示开关；
- 文字：toast、状态行和 ARIA live；
- 声音：商务研究产品默认不启用，避免干扰；
- 键盘：焦点环、Enter/Space 选择、Escape 取消、方向键导航。

## 6. 性能门槛

- 输入到视觉反馈 < 100ms；
- hover 不能触发后端请求；
- 动画优先 transform/opacity；
- 目标 60fps，P95 >= 55fps；
- 单次主线程 long task > 50ms 的占比 < 5%；
- 参数拖动时重算放入 Web Worker；
- 低性能设备自动减少阴影、粒子和并行动画。

## 7. 验收

- 节点选择、镜头切换、主体切换和参数更新均有明显但克制的反馈；
- 不出现整屏闪白、突然跳位或无响应；
- 开启减少动态后功能完整；
- 触觉不支持时无错误、无空白设置；
- Playwright 通过动画开始时间、最终状态和 reduced-motion 测试。
