# 03 - UI/UX 与信息架构（v4.2 视觉工作台）

## 1. 结论

应用采用 **Workspace-first / Map-first / Watchlist-first**。首页与公司页不再分离：用户打开产品即进入当前公司的可视化研究工作台。行业是导航与空状态入口，不再是独立卡片首页。

## 2. 一级导航

```text
关系图谱（默认）
关注列表
行业地图
重要变化
已保存视图
来源与依据
设置（次级）
```

最终用户界面不显示 Agent、Gate、reroot、hop、profile、calibration 等工程词。

## 3. 工作台骨架

- 左窄导航栏：稳定一级入口。
- 左上下文栏：Watchlist、行业、最近路径、保存视图。
- 中央主画布：当前主体的商业版图，首屏至少占 60% 可用面积。
- 右上下文详情：选中对象摘要、关键连接、证据状态和动作。
- 底部时间条：历史快照和结构变化。
- 顶部：搜索、探索路径、截至日期、保存、列表替代视图。

## 4. 默认入口

有最近会话则恢复；否则有 Watchlist 时打开未读重要变化最高对象；Watchlist 为空时在画布内显示行业入口与演示。禁止先展示行业卡片仪表盘。

## 5. 主图信息结构

上游在左、主体居中、下游在右；资本与控制在上；政策与风险在下；业务板块围绕主体。采用综合、供应链、资金与交易、股权与控制、业务板块、政策与风险、战略变化等分析视角。

## 6. 递归交互

单击节点仅选择；点击“以它为中心”才切换主体。继承时间、分析视角、过滤、分析偏好和探索路径。支持 breadcrumb、返回/前进、固定、比较、路径、保存和列表模式。

## 7. 复杂度与语义缩放

L0 商业帝国集群；L1 业务/供应链阶段；L2 公司/设施/产品；L3 关系与证据。默认 20-40 条关系，超过阈值分组并解释隐藏数量。

## 8. 详情按需展示

右侧详情包含：对象做什么、与当前主体的关系、最重要的连接、公开披露/多源推断/信息冲突/尚未披露、资料完整度、更新延迟、来源入口。公式和权重进入次级“分析偏好”。

## 9. 可访问性

图谱所有主动作可用键盘；可见焦点；不以颜色作为唯一编码；提供列表/表格替代；不得依赖 hover、右键或拖拽作为唯一方式；动画可关闭。

详见 `docs/18-22` 与 `prototype/index.html`。
