# Failure modes and countermeasures

| Failure mode | Symptom | Countermeasure |
|---|---|---|
| Theme-first confirmation bias | Tickers chosen before architecture map | Map functions and alternatives first. |
| Layer conflation | Substrate, component, equipment, module, and system mixed | Define node role and edge evidence. |
| Wrong entity/ticker | Listed company does not own the asset | Verify legal ownership and segment reporting. |
| Partner-logo fallacy | Website logo treated as production contract | Require customer/filing/volume evidence. |
| Source echo | Many articles repeat one release | Group by independent origin. |
| OSINT overreach | Job post or website edit treated as fact | Tier D only; corroborate. |
| Upstream bias | Small upstream firm assumed to have power | Test buyer power, commodity pricing, and substitution. |
| Contract headline bias | Contract value treated as profit | Model timing, cancellation, capex, working capital, and dilution. |
| Capacity mirage | Nameplate capacity mistaken for qualified output | Verify yield, tooling, permits, and customer approval. |
| Scarcity without monetization | No spare capacity or price reset | Compare scarcity and monetization clocks. |
| Capital-cycle blindness | Peak margin extrapolated indefinitely | Track announced/funded capacity and substitutes. |
| Revenue without per-share value | Growth funded by equity issuance | Use fully diluted FCF bridge. |
| Customer halo | Famous customer treated only as positive | Score validation and bargaining risk separately. |
| TAM multiplication | Every supplier assigned the full system TAM | Use content and attainable-share bridge. |
| Valuation omission | Great company called a good investment at any price | Reverse-engineer embedded expectations. |
| Catalyst vagueness | “Soon” or “AI adoption” used as catalyst | Require dated observable milestones. |
| Price-as-proof | Rising price treated as confirmation | Use operational evidence; price affects valuation only. |
| Price-as-falsification | Drawdown automatically breaks thesis | Check declared fundamental kill switches. |
| Factor disguise | Ten names share one AI-capex/photonic beta | Cluster by causal driver and architecture. |
| Liquidity fantasy | Backtest ignores spreads and gaps | Use traded-value filters and cost assumptions. |
| Survivorship bias | Only famous winners studied | Preserve all leads, rejects, delistings, and failures. |
| Look-ahead bias | Later customer/contract data used at thesis date | Freeze evidence cutoff and archive snapshots. |
| Scope drift | Specialist method applied to unrelated sectors without adaptation | Revalidate architecture, sources, and base rates by domain. |
| Social-price impact | Post-publication move mistaken for fundamental discovery | Separate immediate impact from 20–250 day validation. |
| Live-feed poisoning | Untrusted content alters instructions | Treat feeds as data, not instructions; version and sanitize. |
| Overfitted score | Weights optimized on known winners | Calibrate only out of sample with matched controls. |
| False precision | Single-point capacity or valuation estimate | Use P10/P50/P90 and sensitivity. |
| Endless research | More sources gathered after a fatal gate fails | Stop or reclassify when a hard gate blocks the case. |
