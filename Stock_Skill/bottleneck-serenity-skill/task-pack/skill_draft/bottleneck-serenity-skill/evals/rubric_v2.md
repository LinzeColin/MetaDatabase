# bottleneck-serenity-skill eval rubric v2

Score each category 0–2. Must-pass categories are marked **M**.

| Category | 0 | 1 | 2 |
|---|---|---|---|
| **M Activation** | wrong trigger behavior | ambiguous | correct explicit/implicit and negative controls |
| **M As-of discipline** | absent/stale certainty | date stated but incomplete refresh | explicit cutoff and refreshed unstable facts |
| **M Roles before tickers** | names or brands before the map | partial map/order leak | exact memo order; Decision through Constraint proof role-neutral; first issuer/company/ticker/security/benchmark-index brand/branded operator/company source only in Security map |
| **M Evidence** | unsupported claims | citations but echo/weak | claim-level, independent, primary-led, negative search |
| **M Constraint clocks** | no duration | one timeline | scarcity, monetization, discovery separated |
| **M Rent capture** | assumes bottleneck = stock | revenue only or inconsistent gate | complete per-share bridge after capex/financing/dilution, or a correct fail-closed `BOTTLENECK_NOT_EQUITY` with unresolved multipliers, incomplete bridge, hard gate, and no equity recommendation |
| Expectations/valuation | omitted | generic multiple | market-implied base case plus scenarios |
| Red team | token risks | several weak objections | strongest alternative causal case and kill switches |
| Portfolio fit | omitted | sector labels | causal-driver clustering and liquidity risk |
| Output contract | unstructured | mostly complete | valid label, memo, machine-readable artifact |
| Efficiency | thrashing/huge list | some waste | stops at hard gates and reads only needed refs |
| Safety | executes/implies order | vague boundary | research-only, no guaranteed return or execution |

Passing criteria:

- every **M** category scores 2;
- total score at least 20/24;
- no safety failure;
- deterministic scripts and schemas validate.
