# 下一尽调门禁

## 1. 目标

下一步不是“多看一些资料”，而是以最低成本关闭一个会改变 research-priority 状态的关键缺口。不得用继续写报告替代证据。

## 2. Diligence Card

```text
Candidate / issuer / security:
Current status and E-level:
Critical claim or assumption:
Why it changes the rank:
Required source category:
Method / exact field to retrieve:
Metric, period, denominator and timestamp:
Pass / advance threshold:
Fail / reject threshold:
Inconclusive branch:
Time / data / coordination cap:
MNPI / license / access boundary:
Owner and review date:
```

## 3. 缺口到工作流

| 缺口 | 首选尽调 |
|---|---|
| issuer/ticker/share class | regulator/exchange/company identity check |
| commercial mechanism | industry primary sources + payer/value-chain map |
| exposure attribution | latest filing/segment note/product/geography denominator |
| capture | orders/backlog/revenue/margin/guidance bridge |
| estimates/revision | current consensus/export or disclosed KPI bridge |
| valuation | normalized price/share count/EV/earnings/cash-flow context |
| catalyst | confirmed calendar/source + impact branch |
| downside/falsifier | countercase filing, competitor, cycle or regulation check |
| model implication | route to model/valuation workflow |
| thesis synthesis | route to long/short pitch or memo only after gates |

## 4. 证据强度

- `management said` 是线索，不自动证明 materiality；
- product exposure without denominator 不能证明股票相关性；
- revenue growth without margin/cash-flow path 不能证明 value capture；
- price rally 不能证明 crowding；
- one multiple without period/peer/earnings context 不能证明 cheap/expensive；
- social/AI persona 不代表 consensus 或 positioning。

## 5. 三分支必须改变决定

| 结果 | 必须动作 |
|---|---|
| Pass | 升级指定 claim/E-level 或路由到更深工作流 |
| Fail | REJECT、降级、缩小 exposure 或重构 mechanism |
| Inconclusive | 不升级；只在 VOI 仍高时设计更窄检查 |

没有 fail branch 的“尽调”是信息消费，不是门禁。

## 6. 默认停止

任一即停：数据需要未授权账户/付费墙绕过；疑似 MNPI；provider/period/units 不可核验；关键 exposure 小到无法量化；估值/price 过期；成本超过 cap；结果不会改变状态。

## 7. 完成定义

Card 在执行前写定；结果引用 source IDs；deviation 可见；E-level 由 evidence signals 保守重算；`ADVANCE_RESEARCH` 只触发下游深研，不触发交易。
