# 商业机制、受益路径与发行人敞口

## 1. 不要把五个对象混成“股票机会”

| 对象 | 含义 | 常见误判 |
|---|---|---|
| Signal | 新闻、价格、搜索、政策或管理层措辞 | 热度直接等于机会 |
| Commercial opportunity | 有 payer、价值池、瓶颈与可持续捕获路径 | 只有 TAM 和叙事 |
| Beneficiary pathway | 谁直接/间接受益、谁承担成本 | 全产业链都受益 |
| Issuer exposure | 上市公司具体 segment/product/geography 的证据 | 公司提过关键词即暴露 |
| Equity setup | 当前预期、估值、催化、风险与可交易性 | 好公司等于好股票 |

## 2. 商业机会最小链

```text
Actor/payer -> demand trigger/problem -> budget/value pool
-> bottleneck/constraint -> supplier or solution -> pricing/capture
-> capacity/cost -> orders/backlog/revenue/margin/cash-flow effect
```

每个箭头都要有 Fact、Inference、Estimate 或 Unverified 标签。市场规模不能代替 payer、budget 和 capture。

## 3. 受益路径

至少检查：

- direct beneficiary：直接销售关键产品/服务；
- picks-and-shovels：供应设备、材料、数据、基础设施；
- capacity owner：稀缺产能/许可/网络；
- distributor/integrator：控制渠道、实施或服务；
- substitute/disruptor：替代旧方案；
- cost bearer：增长可能压缩其利润；
- false positive：相关词很多但经济暴露很小；
- laggard：有暴露但无产能、渠道或执行能力。

先画路径再列 ticker。每个候选写 exposure numerator 与 company denominator，避免小业务被大市值/大收入稀释。

## 4. 发行人和证券身份

最少字段：legal issuer、ticker、exchange、security type、share class、ADR/local line、currency、fiscal year end、reporting segment、market-data timestamp。重复上市、壳公司、基金、债券和衍生品不得混入普通股 universe。

## 5. 敞口证据阶梯

由弱到强：

1. 公司/媒体提及主题；
2. 产品、客户或地域有可验证关联；
3. segment 或 exposure 有量化 denominator；
4. orders/backlog/revenue/capex/guidance 可追踪；
5. margins/earnings/cash flow/estimate revisions 有可解释路径；
6. 当前 valuation、expectations 与 catalyst 可判。

1 不能单独升级 E2；2–3 支持 exposure attribution；4–5 支持 commercial capture；6 支持 equity setup。

## 6. What is priced in

不要把“估值高/低”当结论。拆成：

- 市场隐含增长/利润/周期假设；
- consensus 或公开预期（若可得）的时间戳与范围；
- 哪个 KPI/事件会形成 revision；
- 预期已满时需要什么超额证据；
- valuation downside、multiple/earnings 双重风险；
- price action 只能提示 expectations risk，不能证明 crowding。

## 7. 五类假设

| 类别 | 问题 |
|---|---|
| mechanism | 价值池和瓶颈是否真实/持续？ |
| exposure | 发行人是否有足够、可量化的经济敞口？ |
| capture | 敞口能否进入订单、收入、利润或现金流？ |
| setup | 预期/估值/催化是否留下研究空间？ |
| risk | 反证、法规、周期、资本、流动性或治理是否推翻？ |

## 8. Surprise 扫描

主动寻找：主题赢家可能是供应商而非品牌；收入增长但 margin 被竞争/成本吃掉；订单不可取消性弱；capex 受限；政策补贴退坡；客户集中；份额增长已 priced in；最纯敞口公司不可交易；最大受益者未上市；“受益股”只是小业务标签。
