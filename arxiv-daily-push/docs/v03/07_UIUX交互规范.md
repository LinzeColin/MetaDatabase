# 界面与交互规范 V0.3

参考实现：`ADP 原型.dc.html`（六入口 + 六主题）· `新邮件模板.dc.html`。

## 1. 信息架构（六个一级入口）

今天学什么（讲义+回忆+手动状态+迁移+深挖）· 学习队列（五态可编辑）· 前沿雷达（板块一～五切换+债务+弃权）· 证据与纠错（声明分型/矛盾/纠错通知）· 应用与资产 · 系统与来源（运行五态/授权/权重/自迭代/数据安全）。移动端底部四标签：今天/队列/雷达/系统。

## 3. 深度讲义结构（八段，目标系统内学到 80–90%）

人话 → 领域脉络 → 机制拆解（含历史案例）→ 证据与数字 → 反例与边界 → 跨领域连接+≥2 个意外收获 → 可复用方法 → 术语表；句句可溯源；结尾「继续深挖」按钮携带全文上下文 + Owner 深挖指令（全网遍历、深度思考、深度搜索、给我 surprise、详细专业全面深度讲解 + 已学要点）开 ChatGPT 新对话。

## 4. 关键仪式与反馈

按住 0.8s 显影（进度充填 + 每 24% 微振 6ms + 轻 tick）→ 四档自评（按钮标注 FSRS-6 下次间隔）→「已学会」确认。声音 ≤0.05 音量（导航 1.2kHz 短音；显影 523+784Hz；四档 196/294/392/523Hz，忘了低哑、轻松琶音）；振动 8ms 轻点 / [60] 忘了 / [10,30,10,30,20] 轻松；声音与触感独立开关。

## 5. 手动控制（一等功能）

- 状态编辑：今日页 + 队列每行，五态按钮，点已选中撤销；主观标记单独记账。
- 前端调参：参数风洞拖动即所见 → 发起提案 → 回放预览 → 应用回执；每周日自迭代提案走同一 UI。

## 6. 可及性

点击目标 ≥44px；底栏 56px；讲义正文 ≥14.5px、行高 ≥2.0；全中文（专业术语保留原文并即时解释）；键盘 1–4 评分。
