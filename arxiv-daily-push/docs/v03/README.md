## 五层文档职责（本包的组织原则）

| 层 | 回答的问题 | 目录 |
|---|---|---|
| PRD | 为什么做、为谁做、做成什么样 | `01_PRD/` |
| 系统架构 | 系统长什么样：功能、参数、数据、依赖、规则、流程 | `02_系统架构/` |
| 执行计划 | 这一次如何安全地做 | `03_执行计划/` |

| 运维手册 | 交付后你怎样使用和维护 | `05_运维手册/` |

## 推荐读取顺序

1. `01_PRD/PRD.md` — 产品目标、目标用户、用户问题、使用场景
2. `06_需求继承清单.md` — 你的旧 PRD 一条不丢的对账总账
3. `02_系统架构/操作逻辑与流程.md` — 一天一次心跳，五个环节
4. `02_系统架构/配置参数与阈值注册表_v0_3.yaml` — 全系统唯一参数真相（38 个）
5. `03_执行计划/ROADMAP.md` — R0–R6 交付制路线图（每周五见结果）
6. `03_执行计划/追寻目标PROMPT.md` — 每次开发会话置顶的一句话
7. 其余按需查阅；`05_运维手册/RUNBOOK.md` 在部署时读

## 执行代理（Codex 等）的使用约定

- 每次会话：置顶追寻目标 Prompt → 按 `03_执行计划/EXECPLAN模板.md` 填一份本次执行合同 → 只做一个任务。
- 上下文预算 ≤100 KB（本包全部文件合计 <80 KB，特意如此）。
- 完成定义：验收断言绿色 + 在 `04_开发记录/CHANGELOG.md` 追加一行 + 运行落一条 Run Manifest。禁止新增任何其他文档。
- 同一问题修复 ≤2 次，第 3 次强制换方案（回滚 / 降级 / 换成熟组件）。
- 参数只能改阈值注册表，且每次修改必附回放对照。
