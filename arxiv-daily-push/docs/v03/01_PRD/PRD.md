# PRD — 个人前沿学习系统 V0.3

状态：Draft（待 Owner 签署）· 取代对象：ADP-PRODUCT-CONTRACT-V7.2（签署后切换）· 绑定：本包全部文件

## 1. 为什么做（问题）

- 跨学科前沿信息每天产生，Owner 只有有限注意力；现有系统「发现—排名—生成—发送」链路存在，但对「理解—回忆—迁移—纠错—长期保留」没有任何产品合同——**发了邮件 ≠ 学会了**。
- 现有工具割裂：找论文（Semantic Scholar）、读摘要（邮件）、深挖（ChatGPT）、复习（无）、应用（无），Owner 需要在系统内一站学到八九成。
- 治理文书 ≈4.5MB 吞噬每次开发预算，参数上千个自由度不可验证，一个 bug 反复修导致长期无法交付。

## 2. 为谁做（目标用户）

单一 Owner：跨领域兴趣（人工智能、半导体、生物科技、医疗、金融资本、先进制造、能源、数字经济），高中理科基础起步的成年人，时间有限，重视证据与实际迁移，不写代码、只验收结果。

## 3. 做成什么样（产品定义）

> 一个证据可追溯、能把前沿信号转化为「真正学会并能迁移应用」的个人前沿学习系统；网页为主界面，邮件只是镜像通道。

**每日**：1 篇深度讲义（人话→脉络→机制→证据→边界→连接→意外收获），系统内学到 80–90%；「继续深挖」按钮带全文上下文 + Owner 深挖指令（全网遍历、深度思考、深度搜索、给我 surprise、详细专业全面深度讲解）打开 ChatGPT 新对话。
**每周**：跨板块前沿雷达 + 知识债务 + 权重自迭代提案。
**每月**：个人画像更新（兴趣漂移、难度、遗忘曲线）+ 前沿地图。
**随时**：主动回忆（按住显影 + 四档自评 + FSRS-6 排程）；任何一篇的状态可手动修改；参数可在前端手动调整（提案→预览→应用）。

## 4. 使用场景（核心旅程，验收以此为准）

- J1 每日专注：系统选择并解释理由 → 阅读讲义 → 主动回忆 → 自评 → 排复习 →（可选）迁移练习。发送不改学习状态；重复运行不重复发送。
- J2 纠错传播：来源出新版/撤稿 → 受影响声明标记 → 讲义与知识项重开 → 强提醒 → 差异对照 → 重新复习。
- J3 参数修改：Owner 在前端改权重/阈值 → 校验 → 30 天回放影响预览 → 显式应用 → 回执；每周日系统反向发起同流程的自迭代提案。
- J4 板块扩展：板块二～四各走「提案 → 接入 → 两周影子 → Owner 确认 → 上板」，随时可停。
- J5 随处访问（可选）：手机打开 home.linzezhang.com 私有入口完成当日学习；断网时本机闭环照常。

## 5. 信息架构（六个一级入口）

今天学什么 · 学习队列（状态可编辑）· 前沿雷达（含板块一～五）· 证据与纠错 · 应用与资产 · 系统与来源（运行真相/授权/权重/自迭代/数据安全）。六主题一键切换：暖纸学习 / 简约专注 / 清新干净 / 炫技 / 宇宙星河 / 森林河流。

## 6. 目标与非目标

**目标**：降低发现与理解成本；提高 7/30 天回忆保留；产生真实迁移与资产；证据、版本、纠错全程可追溯；低维护、零付费依赖、失败关闭的安全运行；任何数据永不丢失。
**非目标（防止范围复活）**：视频/语音/TTS 为零；不做多用户 SaaS；不追热闹度排行；不以邮件数量/点击率/测试数量作为成功标准；3+1 与五封仅保留为模板能力。

## 7. 成功度量

30 天试运行报告四指标：7 天回忆成功率、30 天回忆成功率、复习债务（到期未复习数）、注意力成本（日均学习分钟）；辅以每周弃权质量抽查。文件数量、测试数量、绿色徽章一律不算成绩。
