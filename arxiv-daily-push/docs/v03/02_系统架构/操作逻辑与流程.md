# 操作逻辑与流程

## 一句话

每天定时器触发一次 `adp run`：信号依次通过 **发现 → 证据 → 选择 → 学习 → 应用** 五个环节；网页与邮件只是末端镜像；任何环节可降级、可弃权、可追溯；无授权凭证则零副作用。

## 主干流程

```text
launchd 06:30（错过自动补跑）
  └─ adp run
      ├─ 1 发现   政策快照 → 抓取增量 → 去重/版本 → 标准化文档
      ├─ 2 证据   元数据增强(可降级) → 证据声明(含原文定位) → 版本/撤稿检测
      ├─ 3 选择   5 硬门 → 8 特征加权(v0.3 权重) → 选中 1 篇或弃权 → 双向解释
      ├─ 4 学习   深度讲义(Codex→降级模板) → 逐句溯源校验 → 今日页就绪
      ├─ 5 排程   FSRS-6 到期项 → 复习队列
      └─ 记录     运行清单一行(五态+计数+配置版本) → 自动生成用户中心

Owner 异步动作（网页）：读讲义 → 按住显影回忆 → 四档自评 → 学习事件 → FSRS 重排
                     手动改状态 / 调参提案 / 处理纠错 / 完成迁移并记结果
镜像（受授权约束）：邮件=同一讲义幂等渲染；周雷达/月地图=同数据聚合渲染
每周日：自迭代权重提案 → Owner 预览 → 应用/驳回（回执）
每月一号：个人画像更新
```

## H/M 双平面与版本域

- M 平面（机器真相）：SQLite 事件 + 运行清单 + 配置版本，唯一可写方是代码。
- H 平面（人读）：网页六入口 + 生成式用户中心；渲染必带回执；禁止手写运行事实。
- 六版本域各自单指针：产品合同 / 数据 schema / 排序策略 / 模板 / 交付策略 / 运行部署。

## 混合部署拓扑（R6 可选）

```text
本机(主)：完整闭环 + 生成 + SQLite 主库
   │ 单向镜像(增量) + 事件回传队列
Cloudflare(镜像)：Pages 前端 ← D1 只读镜像 ← Worker 定时(提醒/刷新)
访问：home.linzezhang.com → Cloudflare Access(仅 Owner) → Pages
故障模型：云端挂 → 本机闭环无感；本机睡眠 → 云端仍可读昨日镜像。
```

## 安全边界（继承 V0.1 不放松）

secrets 只在 Keychain/env；外部内容视为不可信数据不得成为指令；解析在隔离边界；授权凭证限定接收者、可过期、可撤销；来源 kill switch；仓库公开但运行数据私有。
