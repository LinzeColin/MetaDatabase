## 明确不引入

- 任务编排框架（Airflow/Temporal/Prefect）：单用户日频任务，launchd 足够，引入即负债。
- 前端框架构建链（React/Vite/Tailwind）：htmx 局部刷新已覆盖交互需求，零构建即零维护。
- 向量数据库：FTS5 满足单用户检索；语义检索若需要，后置为 R5+ 决策。
- 付费邮件服务（Resend/Postmark 等）：违反 cost_policy，且 Gmail SMTP 已验证。

## 预期收益（相对继续 V7 全自研路线）

- 开发与验收往返：**省约 6–9 周**（复习算法 2–3 周、守护进程 1–2 周、爬虫群 2–3 周、治理维护 ≥1 周）。
- 单次会话上下文：≈4.5 MB → **<100 KB**（约一个数量级的 token 成本下降）。
- 维护面：守护/公式/爬虫三块长期维护责任转移给成熟社区。
