# 业务规则与系统不变量

## 十条系统不变量（任何任务不得破坏，进入测试保护）

1. 发送事件不得改变学习状态（发了邮件 ≠ 学会了）。
2. 「已学会」必须引用一次不可变的主动回忆事件。
3. 「已掌握」需两次间隔回忆 + 一次真实应用证据；手动标记单独记账，不冒充证据。
4. 纠错/撤稿必须找回受影响的声明、讲义、知识项与掌握度并强提醒。
5. 无 Owner 签发的授权凭证，任何真实发送/常驻一律失败关闭。
6. 同一防重号永不产生第二次发送、学习完成或应用记录。
7. 对外每句事实可追溯到声明与原文定位。
8. 六个版本域（产品合同/数据/排序策略/模板/交付策略/运行部署）各自只有一个当前指针，互不代替。
9. 人读页面不得手写运行事实，一律由运行记录渲染并带回执。
10. 每次排序保存参数、特征贡献与硬门记录，可整体重放。

## 业务规则

- **弃权**：最高分 < 弃权线（当前 55）当日不发；弃权是一等决策，理由进周报；候选顺延入池不丢失。
- **手动状态编辑**：Owner 可将任何条目改为 未学习/学习中/已学会/需复习/已掌握；记录为主观标记事件、可撤销；与回忆证据分开统计；纠错提醒不受手动标记豁免。
- **参数变更**：唯一入口是阈值注册表；路径为 提案（手动或每周自迭代）→ 30 天回放影响预览 → 显式应用 → 回执；无回放对照不得合入；随时可回滚到任一历史版本。
- **自迭代边界**：系统只能「提案」，永远不能自行应用；单次每权重最大步长 ±2；Owner 驳回三次的方向本季度不再重复提案。
- **来源晋升**：接入 → 两周影子（零主干干扰）→ Owner 预览确认 → 上板；连续 3 天失败自动停用并通知。
- **复习保护**：日复习上限 12 项，超出顺延并记知识债务；逾期 >12 项触发保护性复盘。
- **数据**：事件只增不删；删除=标记；每日备份/每周快照/一键导出。
- **修复熔断**：同一问题修复 ≤2 次（各 ≤半天），第 3 次强制换方案：回滚 → 降级交付 → 换成熟组件；连续两个周五同功能降级则重新设计。
