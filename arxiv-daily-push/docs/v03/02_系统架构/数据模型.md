# 数据模型（本地 SQLite WAL+FTS5；R6 后镜像 Cloudflare D1——同为 SQLite 语法）

## 原则

1. **事件只增不删**：一切用户与系统行为都是追加的事件行；「当前状态」永远由查询/物化视图得出，可整体重放。
2. **删除只是标记**：任何对象支持 `archived_at` 标记，历史永可找回——任何数据不会丢失遗忘。
3. **每域稳定 ID**：文档、版本、声明、讲义、事件均有不可变 ID；跨表引用只用 ID。

全文检索：documents/claims/lessons 入 FTS5 虚表。

## 备份与迁移

- 每日 `sqlite-utils` 导出 + 文件级备份（保留 30 份滚动）；每周快照到异地（R6 后为 R2）。
- 旧数据迁移：旧 `sent` → delivery 事件（不映射学会）；旧打开/点击 → 弱参与信号；旧评分 → 历史快照表不复算；缺定位的旧声明标 `EVIDENCE_INCOMPLETE`。
- schema 变更走迁移脚本 + 可回滚 + 完整性校验；D1 镜像单向（本地为主，云端只读镜像 + 事件回传队列）。
