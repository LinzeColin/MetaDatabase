# 配置参考

## 1. 阈值注册表（唯一参数真相）

见 `02_系统架构/配置参数与阈值注册表_v0_3.yaml`（38 参数：5 硬门 + 8 权重 + 4 选择 + 21 复习，另含自迭代与板块扩展区）。改动只走「提案→回放预览→应用→回执」。

## 2. owner_controls.yaml（清理后保留的运行偏好）

| 字段 | 当前值 | 说明 |
|---|---|---|
| production_enabled | false | 与授权凭证双闸 |
| production_auto_enable_after_acceptance | **false（必须清除 true 残留）** | R0-3 任务 |
| cost_policy.* | 全部禁付费 | 不变 |
| runtime.deployment_mode | local_native（R6 后 local_native+cf_mirror） | 混合部署开关 |
| intelligence_provider.priority | codex_cli_chatgpt_auth → local_model → deterministic_degraded | 沿用 |
| email.recipients | 你的邮箱 | 单收件人 |
| email.split_mode | **five_independent_messages 残留 → 删除**（模板能力另存） | R0-3 任务 |
| queue.max_active_items / source_share_cap | 10000 / 0.40 | 已并入阈值注册表，此处只读镜像 |
| interest_profile.preferred_industries | 10 项行业 | 每月画像更新可改 |

## 3. 环境变量 / 密钥（只在 Keychain/env）

`ADP_SMTP_*`（发送通道，配合授权凭证）· `ADP_ALLOW_SMTP_SEND`（默认 UNSET；truthy 且无授权凭证时启动即失败关闭）· `OPENALEX_MAILTO`（礼貌头）· `S2_API_KEY`（可选增强）· `CF_API_TOKEN`（仅 R6 部署机使用）。

## 4. launchd / Worker 定时

本机：`com.adp.daily.plist` 06:30 本地时区 + StartInterval 心跳；错过自动补跑。云端：wrangler `crons = ["30 20 * * *"]`（UTC=悉尼 06:30，冬夏令时需人工核对——定时全部 UTC）。
