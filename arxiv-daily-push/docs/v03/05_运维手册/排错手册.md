# 排错手册（按症状查，全部人话）

| 症状 | 最可能原因 | 你该做什么 |
|---|---|---|
| 今天没收到讲义 | ① 当日弃权（最高分<55）② 运行失败 ③ 机器睡着且未补跑 | 看系统页最近一行：弃权=正常特性；失败=点开看降级项；未运行=检查 launchd 心跳 |
| 讲义有但邮件没到 | 邮件授权过期 / 幂等拒发（当日已发过） | 系统页授权卡看状态；回执里有拒发原因 |
| 邮件重复收到两封 | 不应发生（不变量 6） | 记入问题台账并附两封的防重号——这是最高优先级缺陷 |
| 复习队列越来越长 | 连续多日未复习，触发日上限 12 保护 | 先清逾期项；连续 >12 天积压会自动降低当周新讲义难度 |
| 「已学会」变回「需重新确认」 | 纠错传播：所引论文出了新版/撤稿 | 去「证据与纠错」看差异对照，安排重学（这是特性，不是丢数据） |
| 手动改的状态不见了 | 不应发生（事件只增不删） | 学习队列筛「已手动标记」；仍无 → 用导出包查事件流水，记台账 |
| 参数改了没生效 | 只做了风洞试验未「应用」/ 回放对照未通过 | 检查是否有应用回执；无回执=未生效 |
| 云端 home.linzezhang.com 打不开 | Access 未登录 / Worker 或 Pages 故障 | 先本机访问（闭环不依赖云端）；云端只是镜像，稍后自动刷新 |
| 云端数据比本机旧 | 镜像是单向增量，本机睡眠时不推送 | 唤醒本机或手动 `adp mirror push` |
| 增强证据一直是「摘要级」 | OpenAlex/S2 限速或政策变化 → 自动降级 | 看来源政策快照有效期；主干不受影响，可稍后重跑增强 |
| 声音/振动没反应 | 主题内开关关闭 / 浏览器未交互过 / 桌面无振动马达 | 点击页面任意处后重试；振动只在手机生效 |

原则：任何排错先看 Run Manifest 最近一行——五态 + 降级项 + 弃权原因覆盖 90% 的疑问；剩下的看回执与事件流水，**不猜**。
