# ADP v1.2 产品需求与决策

## 1. 产品决策

ADP v1.2 是现有 Cloudflare 产品的增量质量与可靠性版本，不是重新搭建产品。当前线上版本继续是 `0.41.0`；只有本任务包全部阻断门、受控部署和 14 日稳定期通过后，产品版本才成为 `1.2.0`。

唯一代码真源是 `LinzeColin/MetaDatabase/arxiv-daily-push`。CodexProject 中已删除的旧源、V0.1 的死状态列以及 V7.2 的旧本机 SMTP/runner 合同不得反向覆盖本包；V0.1、V0.3、V7.2 继续作为历史谱系或兼容边界保存。

## 2. 目标用户与结果

- 负责人：每天能从中文界面判断今天值得看什么、系统是否健康以及是否需要行动。
- 学习者：看到诚实、可追证的人话说明；英文原文可以查看，但不能伪装成中文解释。
- 运维者：知道当前 build、来源状态、数据新鲜度、错误预算和可回滚版本。
- 开发及验收线程：一次只处理一个任务，能从 Requirement 追到 Acceptance、测试、证据和精确 Subject。

## 3. v1.2 必须交付

1. 按顺序救援 Google News、诊断 stats-gov、通过 PubMed 接入 Science Advances；每项独立 Run Contract。
2. 修复真实英文条目的人话版、移动端四标签导航和可承重视觉/动效回归门。
3. 对齐产品版本、构建身份和 Python `>=3.12` 运行时真相。
4. 建立免费档优先的 15 分钟探针、可用性、新鲜度、RTO/RPO、错误预算和升级提案机制。
5. 在零新增 P0/P1/UNKNOWN、独立验收、canary 和回滚门通过后部署，并完成 14 个连续健康日。

## 4. 非目标

- 不恢复 CodexProject 旧源，不迁往 OVH/Coolify，不重建已删除的 dormant Cloudflare 资源。
- 不引入付费 API、模型权重、自动付费升级、新数据库或新云服务。
- 不重写六主题、不更换 UI 框架、不扩展鉴权、多租户、TAM、价格或计费。
- 不在单个 Run Contract 中同时修改 Google、stats-gov、PubMed 或多个 UI 缺陷。
- 不把 v0.1 的 90 行 `NOT_STARTED` 当作真实完成度，也不修改历史 ZIP。
- 不以 builder 自报、静态字符串断言、绿色 CI 或单张截图替代真实行为证据。

## 5. 成功指标

- 权威任务包七角色唯一，历史 90 个任务、20 条要求、前端合同和两轮验收全部有明确处置。
- Google retry/backoff 正负控、stats 分类、PubMed 解析与限流测试均执行真实实现路径。
- 六主题肉眼可辨；三个视频主题真实播放；reduced-motion 可用；移动端仅四标签；console error 为零。
- 候选相对锁定历史债集合新增 failure/error 为零，且不恢复被禁止的旧根文档。
- 15 分钟探针覆盖率至少 95%，月可用性至少 99.5%，每日管线新鲜度不超过 36 小时。
- RTO 不超过 60 分钟，RPO 不超过 24 小时；部署失败可自动回滚到锁定版本。
- 生产版本完成 14 个连续健康日，期间无开放 P0/P1、无静默零数据和无未解释 UNKNOWN。

## 6. Kill Criteria

出现真实凭据泄漏、交付身份无法绑定、不可回滚迁移、免费档容量无法满足且未批准付费、关键来源静默零数据、视觉门无法阻断负控或独立 verifier 无法复跑时，停止扩展并保持上一接受版本。
