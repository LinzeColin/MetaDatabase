★已知漏洞（全部由独立复核查出，不是我读出来的）★
  · 【跨函数读取抓不到】——最严重的已知洞。`def is_done(rec): return rec["status"]`
    传进一行索引数据，守卫不响。而这【正是】有人会自然写出 T041 陷阱的方式。
    基于名字的 taint 追不进被调函数的形参，除非做真正的跨过程分析。
  · 同样抓不到：row.items()/row.values() 遍历、csv.reader 的位置下标 row[17]、
    拼出来的键 ("sta"+"tus")、pandas。这些较刻意，但确实是洞。
  · 只扫 *.py。已独立确认目前没有非 Python 读者（无 .sh/.yml/.js/.ts 读它），
    所以这是【诚实的范围声明】，不是藏起来的洞 —— 但它是范围。
  · value_cost_scorecard.py 的 card['owner_signoff']['status'] 是【擦边的假阳性】：
    card 确实在 taint 集里，它逃掉只因为接收者是 Subscript 而不是 Name。
    一次无害的重构（sec = card['owner_signoff'] 再 sec['status']）就会让守卫误报。
    真发生了，是【守卫错了】，不是代码错了。
  结论：它抓自然、直接的形状。它【不能】让这一列变得可以安全读取。
