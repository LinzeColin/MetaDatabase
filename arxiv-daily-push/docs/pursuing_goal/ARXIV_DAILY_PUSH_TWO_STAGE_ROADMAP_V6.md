> 适用基线：V5 文本专注版；视频、TTS、音频、媒体渲染及其测试/存储/调度要求全部为 0。
> 任务编号规范：`S{Stage}P{Phase}T{Task}`，例如 `S1P1T01`。

## 2. Gate 与停止状态定义

### 2.1 正常晋升规则

```text
Task Exit Gate 通过
→ 更新治理记录与真实证据
→ Phase Integration Gate 通过
→ Stage Gate 通过
→ 才允许进入下一层级
```

任何 Task 均不得以“代码已写”“单元测试通过”替代真实 Exit Gate。

### 2.2 全局 Hard Stop 代码

| 代码 | 含义 | 处理 |
|---|---|---|
| `R-CONFLICT` | Prompt、Master、仓库事实或冻结需求冲突 | 进入 `BLOCKED_REQUIREMENT_CONFLICT`，只提交冲突矩阵和一个待确认问题 |
| `RES-PRE630` | 2026-06-30 前触发重资源任务 | 立即停止并标记 `DEFERRED_RESOURCE_GATE_2026_06_30` |
| `RES-GATE` | 新电脑 RAM/磁盘/网络/权限不满足最低运行门禁 | 进入 `BLOCKED_OWNER_ACTION`，只列最小一次性动作 |
| `SRC-POLICY` | 来源条款、许可、robots、官方性或免费条件不明确 | 不得采集或晋升，保留明确阻断状态 |
| `DATA-LOSS` | 迁移、恢复或回滚存在数据丢失/不可逆风险 | 停止写入，恢复到最近一致检查点 |
| `G-DRIFT` | 治理文件、生成文档和事实源不一致 | 不得进入下一 Task |
| `TRACE-FAIL` | Requirement/Feature/Task/Config/Function/Test/Artifact/Evidence 链不完整 | 补齐追踪后再继续 |
| `CFG-FAIL` | 权重、队列、来源或冻结参数校验失败 | 拒绝加载生产配置 |
| `DET-FAIL` | 同输入同配置结果不确定或重跑重复 | 停止晋升并修复幂等/排序 |
| `EVIDENCE-FAIL` | 关键 Claim 无原始证据、法律状态不清或反证审查失败 | Fail closed，不生成正式报告 |
| `EMAIL-DUP` | 同一 Run 重复发送或状态不可追踪 | 停止邮件发送并执行幂等恢复 |
| `REPLAY-FAIL` | 历史重放不独立、发生未来信息泄漏或产物缺失 | 整个重放门禁失败 |
| `LIVE-FAIL` | 真实自然日未完成、真实邮件缺失/重复或恢复失败 | 不得宣布生产通过 |
| `BOARD-FAIL` | 板块必需来源、报告、评分、队列或邮件合同不完整 | 板块保持 Shadow/降级，不得生产 |
| `OPS-FAIL` | 锁、heartbeat、补跑、备份、恢复或资源保护失效 | 停止长期调度 |

### 2.3 缺陷门禁

- P0：数据损坏、秘密泄漏、错误邮件、错误法律状态、生产不可恢复。任何 P0 立即停止。
- P1：核心流程错误、静默丢失、重复 Canonical、错误排序、关键 Claim 无证据。P1 未清零不得晋升。
- P2/P3：可进入 backlog，但必须有原因、负责人、版本和计划，不得静默忽略。

## 5. 时间线与关键路径



## 6. 来源和板块晋升规则

每个新增来源严格使用：

```text
DISABLED
→ TEST
→ REPLAY_PASSED
→ SHADOW
→ BOARD_PRODUCTION
→ FULL_PRODUCTION
```

来源级最低 Gate：fixture、30 日明确终态、48 小时 Shadow、幂等、去重、版本、许可、成功或无更新率目标 ≥95%、静默丢失 0、P0/P1=0。

板块级最低 Gate：必需来源已晋升或有批准降级、板块 30 日重放、连续 2 日候选运行、报告/Claim/邮件合同、排名/队列/淘汰可解释、既有生产无回归。
