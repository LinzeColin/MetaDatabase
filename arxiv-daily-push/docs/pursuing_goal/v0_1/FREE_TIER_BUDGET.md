# FREE_TIER_BUDGET · 免费档硬预算（DIR-007）

> **Owner 2026-07-16 硬指令（开启 R2 时）**：整个 ADP 与**所有 repo 仓库**绝不允许超过 Cloudflare 免费档额度。
> 任何 agent **不得让用量超额**，**不得更改或提升额度限制**，除非 **Owner 再三确认并明确授权**。
> 必须**持续监控并更新**用量；**用超即停**（fail-closed）。本文件是 DIR-007 的执行细则，优先级同 OWNER_DIRECTIVES。

## 1. Cloudflare 免费档额度（当前基准）

| 资源 | 免费档额度（每月） | 监控字段 |
|---|---|---|
| **R2 Storage** | **10 GB** | 累计对象字节 |
| **R2 Class A operations**（写/list：PutObject/ListObjects…） | **1,000,000** | 每月写/list 次数 |
| **R2 Class B operations**（读：GetObject/HeadObject…） | **10,000,000** | 每月读次数 |
| Cloudflare 套餐 | Free（FACT-013） | 不得升级为付费，除非 Owner 三次确认 |
| D1 / Workers / 其它 | 各自免费档 | 同样不得超 |

> 额度以 Cloudflare 官方免费档为准；若官方调整，先更新本表并告知 Owner，**不擅自按新上限扩用**。

## 2. 硬规则

1. **不得超额**：任何写入/存储/请求在会导致超过上表任一额度时，**必须停止**（fail-closed），不得继续。
2. **不得代改额度**：任何 agent 不得开启付费、提升 limit、绑定信用卡、升级套餐——**除非 Owner 再三确认且明确授权**。
3. **持续监控**：R2 写入路径必须内置**预算检查**（写前核对累计用量 vs 上表），并在证据中报告当前用量。
4. **保守边界**：接近额度（如 ≥80%）应告警并优先停/降速，不赌临界。
5. **适用范围**：不止 R2——**ADP 全系统 + 所有 repo 仓库**的任何 Cloudflare/云资源用量都受本预算约束。

## 3. R2 写入路径的内置预算停（S2 起）

- R2 dual-write（T022 起）**必须**：
  - 写前：估算本次对象字节 + 本月累计，若将超 10 GB → **不写、记 quarantined_over_budget**。
  - 写/list 计数：本月 Class A 累计接近 1,000,000 → **停写**。
  - 读计数：本月 Class B 累计接近 10,000,000 → **降级/停读**。
- feature flag 默认关；开启后仍受本预算硬停约束。
- 每次 R2 相关任务的证据必须报告：当前 storage 用量、当月 Class A/B 计数、距额度余量。

## 4. 监控与更新

- 每次触碰 R2 的任务：用 `wrangler r2`（只读）或用量 API 采集当前用量，写进证据 `cost_value.json` 的 `free_tier_usage`。
- 发现接近或超额：**立即停**并告知 Owner，不自行扩额。
- 本文件的额度数字若因官方变更需更新：更新后仍**保守用**，并在 events.jsonl 记录来源。

## 5. 与其它规则的关系

- DIR-005（收益覆盖成本）之上再加 DIR-007（免费档硬顶）：**即使 ROI 为正，也不得超免费档**，除非 Owner 授权改额度。
- 违反 DIR-007 属 Stop-the-line 级事故。
