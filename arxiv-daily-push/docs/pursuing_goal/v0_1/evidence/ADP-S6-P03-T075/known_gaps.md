# Known gaps · ADP-S6-P03-T075｜Shadow 主题加速与中央—地方扩散预测

诚实披露本任务**未做**、**边界**与**权衡**（不夸大 shadow 预测能力）。

## 范围与语气
- **SHADOW / 库层**：两个 pilot 预测器在 dev/shadow 环境计算，**不触生产**——无 worker 改动、无 D1/R2 写、无 cron、无云成本；实时 build `b189d3cc0703`（== T040）不变。**不是**上线预测服务。
- **无确定语气是硬约束（两层）**：①**主防线**=surfaced 文本只由硬编码对冲模板 `phrase()` 生成（「估计概率约为 X%（…非确定预测）」），且概率被 clamp 到 [0.02,0.98] 严格落在 (0,1)，故 0/1 永不 surface；②**兜底**=`assert_no_deterministic_tone` 拒绝一组确定措辞（必然/必定/必将/一定/势必/注定/百分之百/毫无疑问/无疑/definitely/certainly/inevitable/assured/… 等）与过度自信的 0/1 概率。词表非穷举（自然语言无法穷尽），故**语气安全主要靠 ①**（模板化输出），词表是纵深防御第二层。**这不代表模型很准**——它代表我们**从不把预测说成必然**。

## 模型的边界（不夸大）
- **特征极简**：模型只条件于 `sign(support - counter)`（净支持信号的正负桶），Laplace 平滑向全局率收缩。这是一个**刻意可解释的桶基线之上的窄条件模型**，不是复杂时序/因果模型。它证明「净支持信号是领先指标」这一条能**打败无信息的无条件基线**，仅此而已。
- **基线是无条件基率**：`baseline_predict` 用训练集全局发生率对所有 val 样本给同一个概率（忽略信号）。这是预测校准里的标准 climatology 基线，非稻草人。
- **模型不完美、可能输，赢在 aggregate 而非每个案例**：**验证窗口是刻意含噪的**——每个 val 窗口都恰带 1 个 impurity（强支持却没发生 / 弱支持却发生了；6/6 窗口，复核逐窗口审计确认非 0），所以模型在个别案例上会预测错。它在**窗口聚合 Brier** 上稳定优于基线（ACCEL 0.163/0.179/0.161 vs 0.250/0.258/0.250；DIFFUSION 0.169/0.146/0.137 vs 0.278/0.262/0.257），**不是**在可完全分离的玩具上。verifier 的**腐化信号探针**证明：把 val 信号反转后模型 Brier 反而恶化到 0.46–0.56（远差于基线）——即模型**真的依赖信号**，信号错了就输。
- **优势来自信号，非 harness**：verifier 的**真·特征消融**控制把 support/counter 全置 0（净信号消失），模型**逐字节塌回基线**（ablate==base，打平、不 beat）。这证明「三窗口优于基线」来自 support/counter 信号本身，而非回测框架。
- **fixture 是构造的 2016+ 事件链**：`build_diffusion_predictor.py` 用**构造**的月度案例演示方法。**两个 pilot 用不同 fixture**（不同基率 ~0.5 vs ~0.6、不同信号强度、不同 val 案例数 5 vs 6、不同 horizon 90 vs 180），故 Brier profile 相异、是**两个独立演示而非同一模式换标签**。它**不是**真实 arXiv/A0 抓取语料的回测——真实语料回测须在 S6 后续/生产阶段以真实 settled outcomes 跑。此处证明的是**方法**（时间不交叉滚动回测 + 预定义目标 + 对冲语气 + 特征打败无条件基线 + 含噪验证），非**生产精度数字**。

## 两个 pilot 的定义（预定义、可结算方向）
- **ACCEL-PILOT**（horizon 90d）：研究主题是否在 90 天内加速。support=近月发表计数上升的月数；counter=饱和/下降信号数。
- **DIFFUSION-A0-A1**（horizon 180d）：中央 A0 政策是否在 180 天内扩散到某省（A1）。support=已回声/引用该 A0 政策的省份数；counter=被取代/矛盾的中央事件数。
- 两者 horizon **预定义**在 `PREDEFINED_TARGETS`，回测前固定；非预定义目标（post-hoc）被 `is_predefined` 拒。**A2（地市）扩散**方向本任务未建 pilot——A0→A1 先行，A1→A2 留待后续（T054/A2 注册表已在 S4 建，但扩散预测目标未定义）。
