# Known gaps · ADP-S5-P02-T062

- **行级粒度（非字符级）**：diff_payload 在**去噪正文的行级**上定位 add/delete/modify（每条带行号 + 原文），足以让用户"看到原文位置"。行内字符级高亮（intra-line diff）留待版本 UI 消费层，本任务不需要。
- **difflib 对齐启发式**：`SequenceMatcher.get_opcodes()` 决定哪些行配对为 modify、哪些拆为 delete+add。对相邻/重排/整块重写，某些配对是启发式而非唯一解——例如整块重写时 difflib 可能把**无直接对应**的行配成一对 modify（A→C、B→D），而非语义上更准确的 delete+add。**每条变更展示的行号与旧/新原文都真实无误、可定位、绝不凭空捏造**；但"modify vs delete+add"的切分依赖 difflib 语义，因此对整块重写，**op 的类别标注不保证语义唯一**（这是行级 diff 的固有性质，行内字符级对齐留待 UI 层）。
- **as-of 依赖 observed_at 的解析日期**：as_of 用 T056 的解析日期（非字典序）比较，`observed_at` 缺失时回退 `doc_date`；两者皆缺则该版本不参与 as-of（解析器对 None 保守处理）。真实生产版本链的 observed_at 由摄取时的抓取时间戳提供。
- **噪声规则边界（诚实披露，已验证）**：模板噪声由 T026 `version_engine.NOISE_*` 规则界定，这是**单一事实源**，本读层原样继承、**不在 T062 内分叉**。两类规则的**保护力不同**，本任务已逐一实测（见 verifier §2b）：
  - **`发布于/更新于/编辑于/发表于`族**：有**前导数字**内容护栏（`\d[...]` 锚），因此以"发布于"开头但**含真实内容**的句子（如"发布于顶级期刊：Nature 发表了新的政策评估。"）**不被误删**——实测保留、并在 diff/replay 中出现（verifier §2b(i)）。
  - **`来源/责任编辑/编辑/审核/校对`族**：规则 `^\s*(?:责任编辑|编辑|校对|审核|来源)\s*[:：].*$` 为**整行**编辑署名/转载来源 chrome，**无内容护栏**——`来源：X` / `责任编辑：Y` 整行按噪声删除，**无论冒号后是什么**。因此**仅发生在此类行内**的变化，按 T026 定义**不属于实质变化**，本读层**不予显示**（显示反而违反"模板噪声不显示"）。这与 ADP 架构一致：转载/来源署名本就是 chrome（参见 T038 媒体线索→官方原文 resolver）。verifier §2b(ii) 已把此边界做成**可判别负控制**（`来源：甲省→乙省` 的仅署名变化 → diff.changed=False、timeline 不加版本）。
  - **是否应把 `来源` 溯源重新归类为实质字段**属 **T026 契约范围**（会影响 T057/T066 等下游），如需变更须走 T026 自己的门，**不在本任务内静默修改**——此处仅**诚实标注**，供 Owner/后续任务决策。
- **NOT_DEPLOYED**：本任务是**只读 API 库层**，不接 worker/cron/D1/R2，不改生产版本链数据。真实生产版本链接入与版本 UI 由后续部署阶段（各自的门）负责；本 API 为其提供确定性底座。实时无回归（live build_id b189d3cc0703 == T040）。
- **后继依赖**：T066 Watchlist / Change-only Digest 依赖本任务的 diff_payload（只推实质变化）与 as_of（点时间状态）；T063-T065 研究元数据/引用关系在其之上扩展。
