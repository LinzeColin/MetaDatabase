# Known gaps · ADP-S2-P02-T026

- **NOT_DEPLOYED（任务边界，非缺陷）**：`version_engine.py` 是纯规则/工具，**未接入 worker/D1 写入路径**，未落生产。把版本决策接到 T025 的 `cn_document_versions`（推进 `current_version_no`、真正 INSERT 新版本）属于后续 S2/云端接线，届时须各自过 gate + 六主题基线复验。
- **噪声规则非穷尽**：`NOISE_RULES.md` 覆盖常见中文站点样板（页脚/导航/分享/计数/相对时间/cookie/广告/版权/ICP）与通用追踪参数。新站点的新样板可能残留 ⇒ 被保守地判为「实质变化」而多产生一个版本（偏安全：**宁可多留版本，不吞真实修订**）。扩展方式：向规则列表追加保守规则 + 补 fixture。
- **doc_date 不入触发签名**：纯发布日期变化（常是相对时间回流）不单独触发新版本；真正再版通常同时改正文/状态。若某源存在「仅更正日期、正文一字不改」的合法再版，当前会被跳过——可配置为把 `doc_date` 纳入签名，但会增加版本抖动风险，故默认排除并在此登记。
- **body_hash 依赖上游正文质量**：`strip_noise` 作用于传入的 `body`（+可选 `raw_html`）。若上游把导航/正文混在同一未标注块里且无样板 token，规则无法区分——需要更强的正文抽取（属 factsheet/抽取任务范畴，不在本任务）。
- **附件语义**：附件变化用 `attachments[].sha256` 集合比较，与 R2 内容寻址键（T021/T022）对齐；本任务用 fixture 哈希占位，未强校验 R2 键存在性（留待快照/恢复任务 T027+）。
- **状态取值未枚举约束**：`status` 归一化为小写字符串比较；合法状态集（active/superseded/withdrawn…）的强枚举在 T025 schema 层未约束、此处亦未强制，留作后续内容合同细化。
