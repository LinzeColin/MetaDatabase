# 实现前检查模板

## 允许读取

仅根治理、arxiv-daily-push 当前 Task 相关文件和本任务包。禁止扫描无关项目。

## 允许修改

每个 Task 卡显式列出；公共 Schema、三基生成器、owner controls、邮件编排分别由对应 Task 独占。

## 基础检查命令（执行前先确认仓库实际存在）

```bash
python3 scripts/validate_project_governance.py --all --semantic --drift-report
python3 scripts/validate_information_quality.py --all --fast --fail-on-error
python3 scripts/generate_governance_dashboard.py --write
```

项目测试按当前 Task 选择 focused unittest，不默认全仓扫描。

## 回滚

- 配置迁移保留旧 config_version 和反向转换；
- 数据库迁移提供 down 或恢复到最近一致备份；
- 邮件迁移先 dry-run，生产发送保持 fail-closed；
- 旧五封配置在迁移完成前只读兼容，不能并行发送；
- V7 合同变更通过 revert 当前 Task commit 和恢复前一合同哈希回滚。
