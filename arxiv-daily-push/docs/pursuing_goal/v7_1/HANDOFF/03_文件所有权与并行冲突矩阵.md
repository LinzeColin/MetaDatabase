# 文件所有权与并行冲突矩阵

| 线程 | 独占/优先文件 | 可并行 | 禁止同时修改 |
|---|---|---|---|
| S2PAT05 治理 | product/requirements/roadmap/stop codes/三基生成器 | 来源 fixture | runtime/storage/SMTP 实现 |
| S2PMT01 安全 | trust policy、content schema、URL/依赖门 | 来源 adapter | 同一 EvidencePacket Schema |
| S2PMT02 原子 IO | runtime backup/restore、artifact writer | UI 静态页面 | S2PMT03 同时改 storage migration |
| S2PMT03 并发/outbox | state machine、DB migration、SMTP/outbox | UI 消费已冻结 API | S2PMT02 同时改同一 migration |
| S2PMT04 生命周期 | scheduler/runtime/cache | 邮件模板 | S2PMT03 同时改 lease contract |
| S2PMT05 测试 | tests/load/chaos/e2e | 各模块修复后并行场景 | 不得改生产代码来“让测试通过”而无单独 PR |
| S2PMT06 UI | 用户中心、view generator、safe action layer | 压力测试 | 不得直接写 DB 绕过 state API |
