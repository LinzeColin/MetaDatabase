# 审查轨 C｜页面布局、交互、功能连接与架构

## 结论

本轨发现 **15** 项：P0=0、P1=9、P2=6、P3=0。P0 全部阻塞生产/最终集成；P1 阻塞受影响范围。

## 审查范围

覆盖中文用户中心、四查三基、导航、状态反馈、配置修改、错误恢复、功能追踪、可访问性、邮件 UI 与代码/功能架构连接。

## 独立方法

从 Owner 核心旅程出发，不以目录存在代替可操作性；先判断信息架构和状态动作，再评估视觉与排版。

## 逐项整改要求

### C-001｜当前仓库未落地唯一中文 00_用户中心首屏

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** static_confirmed；当前项目仍以 config/docs/governance 和英文 Owner 文档为主；V7.0 只是待合并合同
- **影响：** Owner 无法在一个入口确认状态、要求、功能、验收和下一步。
- **建议修复：** 00_开始这里.md 作为唯一首屏，展示健康、当前 Task、阻塞、下一次运行、四个查看入口和只改这里。
- **必须测试：** 从项目根两次点击内完成所有常用操作
- **归属 Task：** `S2PIT01`

### C-002｜总控台偏配置容量，缺真实排队/讲解/发送/复习/行动数量

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** static_confirmed；当前 Owner Console/Model And Queue 快照主要显示 max_active_items 等
- **影响：** 无法判断系统真实工作量、积压和交付效果。
- **建议修复：** 从数据库生成 real-time counts、oldest age、SLA breach、today delta；配置值和实际值分栏。
- **必须测试：** 各分项与总数守恒；空/延迟/失败状态正确显示
- **归属 Task：** `S2PIT02`

### C-003｜四查视图缺统一 freshness、事实源与漂移状态

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；V7.0 只定义文件名和字段方向
- **影响：** 四页可能不同步，Owner 无法知道数据截至何时、是否陈旧。
- **建议修复：** 每页显示 generated_at/data_as_of/source_revision/contract_hash/staleness；只从同一 query layer 生成。
- **必须测试：** 故意制造漂移/过期时 CI 和页面同时报警
- **归属 Task：** `S2PIT05`

### C-004｜缺少一致的全局导航、面包屑、返回和关联跳转

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** design_gap；任务包未定义 navigation contract
- **影响：** 文件虽集中但仍需记路径，复杂状态之间无法自然追踪。
- **建议修复：** 所有人类页面顶/底统一导航；每个对象链接到来源→Claim→报告→邮件→复习→行动→ROI。
- **必须测试：** 自动链接检查无断链；核心旅程点击数达标
- **归属 Task：** `S2PMT06`

### C-005｜错误和阻塞信息缺少恢复动作、负责人和安全重试入口

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；现有 blocking_reasons 多为纯文本
- **影响：** 用户知道失败但不知道该改什么、是否可重试、会不会重复发送。
- **建议修复：** 错误卡统一：code/severity/impact/owner/first_seen/last_seen/retry_safe/CTA/runbook/evidence。
- **必须测试：** 每个 P0/P1 错误至少一个可执行恢复动作或明确人工门
- **归属 Task：** `S2PMT06`

### C-006｜修改配置缺少预览、diff、校验、影响分析和一键回滚

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；“一改”仅定义编辑目录，没有交互保护合同
- **影响：** Owner 修改权重、时间或来源后可能直接破坏运行且难以定位。
- **建议修复：** Edit→Validate→Preview diff/impact→Apply→Audit→Rollback；配置编译产物不可手改。
- **必须测试：** 非法配置无法应用；应用后可回滚到上个 revision
- **归属 Task：** `S2PMT06`

### C-007｜用户控制修改缺少 append-only 审计历史

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；V7.0 未定义 config revision ledger
- **影响：** 无法解释结果变化来自模型、数据还是配置修改。
- **建议修复：** config_revision(id, actor, diff, reason, validated_at, applied_at, rollback_of, contract_hash)。
- **必须测试：** 每次有效修改都有 revision；结果产物记录使用的 revision
- **归属 Task：** `S2PMT06`

### C-008｜队列和账本缺搜索、筛选、排序、导出与下钻规范

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** design_gap；四查字段规范未覆盖操作能力
- **影响：** 数据增长后 Markdown 全量平铺不可用，无法定位逾期或失败项。
- **建议修复：** 按状态/板块/来源/日期/严重度筛选；CSV/JSON 导出；详情页；固定常用视图。
- **必须测试：** 1万项队列下首屏仍只显示摘要和 Top 异常，导出完整准确
- **归属 Task：** `S2PMT06`

### C-009｜全量参数可见但缺渐进披露和变更影响解释

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** design_gap；当前模型参数文件已有数百参数，用户容易过载
- **影响：** “全部可见”可能演变成不可操作的大表。
- **建议修复：** 关键参数首屏；高级/工程参数折叠；每项显示影响、范围、默认、回滚值和关联测试。
- **必须测试：** 关键参数≤20 项首屏；全部参数可检索且无遗漏
- **归属 Task：** `S2PIT03`

### C-010｜功能→Task→测试→运行证据在 UI 中没有可点击追踪链

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；CSV 追踪矩阵存在，但尚未成为 GitHub 人类页面的一部分
- **影响：** Owner 无法直接判断某功能是否真的完成，而不是只写了文档。
- **建议修复：** 功能清单每行加入 requirement/task/status/test/evidence/last_verified；开发记录反链；证据可点击。
- **必须测试：** 每个 required feature 追踪覆盖 100%，无孤儿 Task/测试/证据
- **归属 Task：** `S2PAT05`

### C-011｜旧 B1–B5/五邮件/英文名称与新 3+1 合同并存

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** global
- **证据状态：** static_confirmed；当前 stage1_b1_report.py 与 owner_controls 仍采用旧命名/禁止 ROI
- **影响：** 不同 agent 会按不同产品模型开发，产生不可兼容接口。
- **建议修复：** 单一 migration matrix；旧 ID 标 legacy_alias；用户界面只显示新中文名；CI 禁止新代码再引入旧合同。
- **必须测试：** 全仓扫描旧标识仅允许在 archive/compat 列表出现
- **归属 Task：** `S2PAT05`

### C-012｜缺少手动重试、取消、重新排队、跳过和重新生成的安全交互

- **严重程度：** P1（高风险功能/可靠性级）
- **是否阻塞合并：** scope
- **证据状态：** design_gap；控制面目前主要为只读 Markdown/配置
- **影响：** 发生失败或误选时只能改数据库/文件，易制造重复副作用。
- **建议修复：** 所有动作先 preview；显示幂等键与影响；危险动作二次确认；只允许状态机合法转换。
- **必须测试：** 重复点击不重复发送；非法状态动作被禁用并解释原因
- **归属 Task：** `S2PMT06`

### C-013｜缺少无障碍、响应式和主流邮件客户端兼容验收

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** not_verified；HTML 使用简单内嵌样式，未见 a11y/client matrix
- **影响：** 移动端、暗色模式、屏幕阅读器或 Outlook 等可能难读/错位。
- **建议修复：** 语义标题、alt、对比度、可触控尺寸、纯文本等价；Litmus 类矩阵或本地 snapshot。
- **必须测试：** WCAG 基础检查；Gmail/Apple Mail/Outlook 关键模板截图或结构测试
- **归属 Task：** `S2PMT06`

### C-014｜缺少 loading/empty/degraded/stale/partial-success 的完整反馈状态

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** design_gap；页面规范偏“成功数据表”
- **影响：** 数据为空或部分来源失败时，用户无法区分“尚未运行”“没有更新”“系统出错”。
- **建议修复：** 五态设计；每态说明原因、数据截至、下一动作；M1-M4 部分成功可视化。
- **必须测试：** 所有空态和降级态有 fixture/snapshot
- **归属 Task：** `S2PMT06`

### C-015｜用户反馈尚未形成可见的排序/画像/内容改进闭环

- **严重程度：** P2（中风险质量/维护性级）
- **是否阻塞合并：** no
- **证据状态：** design_gap；合同定义反馈按钮，但未定义反馈后状态与模型变更可解释性
- **影响：** 用户点击“太浅/无关”后不知道系统是否学习，长期信任下降。
- **建议修复：** feedback event ledger；展示已采纳/待样本/不采纳原因；模型参数变更需最小样本与回滚。
- **必须测试：** 反馈事件可追踪到后续排序/内容变化或明确无变化原因
- **归属 Task：** `S2PMT06`
