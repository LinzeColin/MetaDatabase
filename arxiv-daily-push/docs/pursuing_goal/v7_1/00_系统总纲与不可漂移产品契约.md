# arXiv Daily Push V7 系统总纲与不可漂移产品契约

## 1. 北极星

系统的目标不是“采集了多少、发送了几封邮件”，而是：

> 在可审计证据的基础上，以尽可能低的注意力成本，持续为用户产生新的知识、新的技术能力、新的思维框架、新的决策质量、新的行动资产和可核验的经济转化。

## 2. 必须融合的十二项能力

1. 多源采集；
2. 来源健康、许可、版本和去重治理；
3. EvidencePacket、Claim/Evidence 和反证治理；
4. 深度理解与前沿变化量；
5. 个人前沿情报与影响映射；
6. 高密度报告生成；
7. 每日 3+1 邮件发送；
8. 中文人类可读控制、状态和验收；
9. 复习与长期记忆；
10. 行动、实验与能力资产；
11. 预计/实际 ROI 和经济转化；
12. 每周、每月复盘与持续优化。

任何一项不得被建设成与主链割裂的“以后再说”孤岛。

## 3. 统一主链

```text
D1–D4 多源信号
→ RawRecord
→ CanonicalDocument / CanonicalEvent
→ EvidencePacket V2
→ Claim / Evidence / CounterEvidence
→ InsightUnit
→ B1–B6 多标签路由
→ 深度理解与前沿 Delta
→ PersonalImpact
→ ActionPlan / ReviewCard / ROIExpectation
→ M1–M4 Report + Email
→ Feedback / Review / Action / Asset / Conversion
→ Weekly / Monthly Synthesis
→ 模型影响预览与 Owner 决策
```

## 4. 四源、三主、三副、四邮

### 四个数据源域

- `D1` 研究、预印本与医学索引；
- `D2` 权威发表、工程开源与产业技术报告；
- `D3` 中国官方政策、法律、科技、产业和经济；
- `D4` 美国官方科技、法律、金融、宏观和技术政策。

### 三个主阅读板块

- `B1` 科学与理论前沿；
- `B2` 工程、产品与产业前沿；
- `B3` 政策、资本与地缘前沿。

### 三个横切副板块

- `B4` 社会与时代变化；
- `B5` 风险、反证与失败；
- `B6` 个人机会、能力与经济转化。

### 每日四封邮件

- `M1` B1 + B4/B5/B6；
- `M2` B2 + B4/B5/B6；
- `M3` B3 + B4/B5/B6；
- `M4` B1–B6 跨板块共振、冲突、时代主线、复习与个人行动组合。

## 5. 不可漂移要求

1. Stage 2 不因 Stage 1+ 优化而暂停。
2. Stage 2 连接器只能输出统一数据/证据契约，不得直接拥有最终邮件模板。
3. 来源域、阅读板块、邮件产品必须使用 `D/B/M` 三套命名空间。
4. B2 已合并“工程与开源”以及“产品与产业”，不得再次拆成两封日邮件。
5. B4/B5/B6 是每封主邮件的强制检查维度，不单独增加日邮件。
6. M4 必须等待 M1–M3 进入明确终态；失败/无更新/降级均要显式。
7. 邮件必须展示预计 ROI、时间成本、置信度和行动路径；禁止用无依据的精确收益承诺。
8. 每周和每月报告必须进入正式 Roadmap、功能清单和验收。
9. 三基文件必须是完整中文内容，不得退化为跳转页。
10. 用户必须看到真实排队、已讲解、已发送、复习到期、行动和转化数量。
11. 当前关键结论、Owner 决策、Roadmap、Stop Gate 和验收证据必须保存在 GitHub。
12. 所有人类界面中文优先；英文仅用于技术标识、代码、路径、API、原题名和精确引用。

## 6. 内容状态机

```text
DISCOVERED
→ NORMALIZED
→ EVIDENCE_READY
→ ANALYSIS_READY
→ QUEUED
→ SELECTED
→ LESSON_COMPLETED
→ REPORT_READY
→ EMAIL_READY
→ EMAIL_SENT
→ REVIEW_DUE
→ REVIEWED
→ ACTION_PLANNED
→ ACTION_COMPLETED
→ ASSET_CREATED
→ CONVERSION_RECORDED
→ MASTERED / ARCHIVED
```

失败/降级状态必须独立存在：

```text
SOURCE_FAILED
EVIDENCE_INSUFFICIENT
ANALYSIS_FAILED
QUALITY_REJECTED
EMAIL_FAILED
REVIEW_OVERDUE
ACTION_ABANDONED
DEGRADED_WITH_REASON
NO_MATERIAL_UPDATE
```

## 7. 用户价值指标

### 领先指标

- 关键 Claim 证据绑定率；
- 前沿变化量和反证覆盖率；
- 金标内容人工评分；
- 邮件阅读/反馈；
- 复习完成率；
- 行动启动率；
- 能力资产形成数。

### 滞后指标

- 实际节省时间；
- 实际节省成本；
- 新增收入或机会；
- 重要决策改善；
- 可复用代码、流程、笔记和框架；
- 预测账本准确率；
- 30/90 日掌握与复用情况。

## 8. 总验收

最终生产验收不能只证明“邮件发送成功”。必须同时证明：

- 数据源和证据可审计；
- 内容达到深度质量门；
- 3+1 邮件无重复且有水位线；
- 用户中心与三基文件中文完整；
- 真实队列和状态数量守恒；
- 复习、行动、能力资产和 ROI 可追踪；
- 周报/月报可运行；
- 30 日重放、2 日真实运行和恢复演练通过；
- P0/P1 为 0。

合同版本：`ADP-PRODUCT-CONTRACT-V7.1`。
