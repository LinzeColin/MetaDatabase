# UI、页面布局与交互信息架构规范

## 首屏

`00_用户中心/00_开始这里.md` 固定包含：系统健康、当前 Stage/Phase/Task、P0/P1、下次运行、今日 3+1 邮件、真实队列、今日复习、唯一下一步。

## 导航

```text
开始这里
├─ 只改这里
├─ 运行与真实队列
├─ 数据源与板块健康
├─ 模型参数与排序
├─ 内容/邮件/复习/行动/ROI
├─ 功能清单
├─ 开发记录
└─ 模型参数文件
```

每页顶部和底部保持相同导航与面包屑；对象详情必须可追踪到来源、Claim、报告、邮件、复习、行动和 ROI。

## 安全交互

所有写操作使用：`预览 → diff/影响 → 校验 → 确认 → 应用 → receipt → 可回滚`。
错误卡必须有 code、严重度、影响、是否可安全重试、Owner、Runbook、证据和 CTA。

## 状态反馈

必须区分：未运行、加载中、无实质更新、部分成功/降级、失败、数据过期。不能用空表代替状态说明。
