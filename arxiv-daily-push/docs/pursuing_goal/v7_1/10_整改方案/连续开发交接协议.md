# 连续开发交接协议

每个 Agent 结束时必须更新 `HANDOFF/00_下一Agent先读.md`，并只留下一个 canonical 下一 Task。

## 必填交接字段

- 当前 contract/roadmap/hash 与 Git commit。
- 已读取、已修改、明确未修改的文件。
- diff summary。
- 所有命令、退出码和证据路径。
- 开放 P0/P1/P2。
- 数据/Schema/配置迁移和回滚状态。
- 当前 runtime/queue/outbox 状态。
- 唯一下一 Task、entry gate、允许读写范围、测试命令、stop gate。
- 可安全并行任务与文件所有权冲突矩阵。

禁止使用“基本完成”“应该通过”“以后补测试”等不可验收表达。
