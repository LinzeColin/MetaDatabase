# 代码级修复规格

## 1. `stage1_runtime.py`

- 用 `atomic_write_json(path, payload)`：同目录临时文件、flush、fsync、`os.replace`、目录 fsync。
- `run_tick` 的锁必须由 `with runtime_lease(...):` 管理，并在 `finally` 释放；锁含 fencing token。
- `restore_runtime_backup`：校验相对路径和 resolved root；复制到临时文件；hash + SQLite `quick_check` + Schema；原库备份；原子替换。
- 辅助文件目标使用安全相对路径或 source path hash，禁止 basename 扁平化。
- Scheduler 使用 XML/INI/PowerShell 结构化生成器，不拼 shell。

## 2. `state_machine.py` / storage

- 历史逐条校验 `from_state == previous.to_state`，末态等于 current_state，时间单调。
- 持久行增加 `row_version`；转换使用 compare-and-swap。
- 运行租约增加 `fencing_token`；所有写入检查 token。
- review/action/asset/conversion 使用正交 ledger，不挤进单线状态机。

## 3. `smtp_delivery.py`

- 业务事务写 `email_outbox`，唯一键 `(cycle_id, product_id, recipient, active_revision)`。
- sender claim outbox；稳定 `Message-ID`；provider response 和 attempt append-only。
- 发送后崩溃由 reconciliation 判断并进入可审计 `UNKNOWN/RECONCILE_REQUIRED`，不得盲重发。

## 4. `lesson.py` / `stage1_b1_report.py`

- Frontstage 所有陈述带 fact/inference/hypothesis/action 类型和证据/前提。
- `lesson_revision_id` 包含 claim 内容、证据 hash、模型/Prompt/合同版本。
- 报告先验证后发布；标准文件 SHA-256；URL scheme/host 校验。
- 取消“禁止 ROI”，改为禁止无假设/无证据/保证收益式 ROI。
