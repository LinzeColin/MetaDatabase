# V7.0 → V7.1 变更摘要

## 新增

- 三条独立审查轨、53 项问题清单、严重度和合并政策。
- 19 个可复现定向探针及当前代码快照。
- `S2PAT05` 审查落库任务。
- `S2PMT01–S2PMT07` 安全、原子恢复、并发/outbox、自动生命周期、压力测试、UI 和独立复审任务。
- 事务发件箱、lease/fencing、M4 cycle watermark、原子文件/恢复、安全缓存和自动关闭合同。
- 中文人类可读的仓库 overlay：用户中心状态、功能清单、开发记录、模型参数新增内容。
- 可执行 `tools/validate_task_pack.py`、合同单元测试和连续交接文件。

## 修改

- 最终生产 Gate 从 `S2PLT04` 后移到 `S2PMT07`；S2PL 只形成集成候选。
- Roadmap 任务依赖显式化；自由文本 Stop Condition 移入 `stop_notes`，机器字段只允许注册 Stop Code。
- requirements 的多 Task 引用从斜杠字符串改为 `task_ids` 数组。
- 产品合同加入不可信内容、安全、副作用、并发、自动运行、压力与 UI 硬门。

## 保持不变

- Stage2 数据源开发继续并行。
- 四数据源域、三主板块、三横切副板块、每日 3+1 邮件、周报/月报、一改四查三基维持不变。
- 旧 V5/V6/V7.0 历史证据不得覆盖。
