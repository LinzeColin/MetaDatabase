面包屑：`00_用户中心 / 当前状态`

本页是 `S2PMT06 / ACC-S2PMT06-UX` 的只读状态入口。它记录 Owner UX、安全操作、状态反馈、错误卡、配置变更预览、队列 drilldown、可访问性和 no-production 边界。

## 状态反馈

| 状态 | 含义 | Owner 下一步 |
|---|---|---|
| `not_run` | 尚无本地证据 receipt | 运行本地验证预览 |
| `loading` | 正在生成只读预览 | 等待或取消预览 |
| `no_update` | 本次无新 eligible items | 查看 unchanged receipt |
| `partial_success` | 非关键板块 stale，但 durable evidence 保留 | 打开 degraded section |
| `degraded` | 非关键依赖不可用，正在显示 fallback evidence | 打开 recoverable error card |
| `failed` | 必需本地验证 gate fail-closed | 先看 runbook，再 retry |
| `stale` | 最新成功 receipt 超过 freshness threshold | 运行 safe refresh preview |

不得用空表代替状态。

## 错误卡字段

每张阻断/错误卡必须包含：`code`、`severity`、`impact`、`owner`、`retry_safe`、`runbook`、`evidence`、`cta`。

## 安全操作链

Owner 可见配置和动作必须经过：`预览 -> diff/影响 -> 校验 -> 确认 -> 应用 -> receipt -> 可回滚`。

适用动作：`retry`、`cancel`、`requeue`、`skip`、`regenerate`。本任务只生成本地 receipt，不写生产队列。
