## 不要误读

| 看到的事实 | 正确解释 | 禁止解释 |
|---|---|---|
| final bundle pass | Stage 2 final bundle 证据链可复审，且已支持 Stage 2 integrated acceptance | S3/DAILY_OPERATION 已进入 |
| P0/P1 zero-proof 为 0/0 | 当前 zero-proof 已证明一级/二级阻断为 0；V7.1 baseline 仍保留为历史基线 | 覆盖或删除 V7.1 历史 baseline |
| S2PLT04 completion report pass | S2PLT01/S2PLT02/S2PLT03/P0/P1 终态依赖已被 completion report 消费 | 可以启用 SMTP、scheduler、Release 或 restore |
| S2PLT04 completion report 的 `S2PLT04_COMPLETED_NO_PRODUCTION_ACCEPTANCE` 状态码 | S2PLT04 completion report 的 `S2PLT04_COMPLETED_NO_PRODUCTION_ACCEPTANCE` 是该 report 写入时的 no-production 状态码 | 不得把该状态码读成当前 Stage 2 integrated acceptance 未写入、未保持或需要回退 |
| Stage 2 integrated acceptance true | 可以说 Stage 2 integrated acceptance 已记录并保持 | 可以说每日后台生产运行已验收 |
| DAILY_OPERATION gate blocked | 当前唯一进入 S3/DAILY_OPERATION 的阻断仍是缺 `FINAL_ACCEPTANCE_BUNDLE/daily_operation_persistent_enablement_authorization.json` | 用 request 包、模板或一次受控真实运行替代持久授权 |
| 一次受控真实运行验收 | 一次受控真实运行验收只说明受控窗口内的发送证据，不会生成或替代持久 DAILY_OPERATION 授权；窗口结束后必须恢复为 `UNSET` 或 false-like | 把一次受控真实运行说成 S3/DAILY_OPERATION 已进入 |
| `codexproject_repo_root_invalid` | verifier 命令不在 CodexProject 仓库根运行，或 `--root` 指错目录 | 把执行根错误当作 S3 授权缺失或生产状态变化 |

不得把 final bundle pass、zero-proof 或 S2PLT04 完成解读成 S3/DAILY_OPERATION 已进入。缺 `FINAL_ACCEPTANCE_BUNDLE/daily_operation_persistent_enablement_authorization.json` 时，`daily_operation_enabled=false` 必须保持，真实 SMTP 发送开启、scheduler 安装开启或 Release 上传完成这类生产启用结论都不得出现。
不得把 request 包、模板或一次受控真实运行当作 `FINAL_ACCEPTANCE_BUNDLE/daily_operation_persistent_enablement_authorization.json`；没有该 live artifact 时，readiness 和 enablement preflight 仍必须 fail-closed。

## 最小复核命令

以下命令必须从 CodexProject 仓库根目录运行；`tools/`、`scripts/` 与 `FINAL_ACCEPTANCE_BUNDLE/` 均为仓库根路径。不要给这些 root tools 追加 `--json`；它们默认输出 JSON。

```bash
python3 -B tools/verify_acceptance_bundle.py --root . --require-zero P0 P1
python3 -B tools/verify_daily_operation_readiness.py --root .; ec=$?; echo "EXPECTED_READINESS_EXIT=$ec"; test "$ec" -eq 2
python3 -B tools/verify_daily_operation_enablement_preflight.py --root .; ec=$?; echo "EXPECTED_PREFLIGHT_EXIT=$ec"; test "$ec" -eq 2
```

预期：第一个命令 `status=PASS`，同时显示 `daily_operation_authorization_ready=false`；后两个命令本体输出 `status=FAIL / exit 2`，随后 `EXPECTED_READINESS_EXIT=2` 和 `EXPECTED_PREFLIGHT_EXIT=2` 且 shell 断言通过。阻断原因为 `persistent_daily_operation_authorization_missing`。

复制命令后还必须核对 JSON 字段：正确仓库根下 `repo_root_valid=true`、`root_validation_errors=[]`、`required_paths_missing=[]`，同时 `authorization_artifact_exists=false`。如果输出 `codexproject_repo_root_invalid`，先回到 CodexProject 仓库根或修正 `--root`，不得把该错误解释成 S3/DAILY_OPERATION 授权状态。
