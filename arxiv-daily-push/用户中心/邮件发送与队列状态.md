## 发送缺口说明

| 项目 | 当前说明 |
|---|---|
| 今日应发送口径 | 必须来自当天正式发送计划或排程配置；没有证据时不得手写任意数字 |
| 今日已发送口径 | 只计算邮件服务器已接受的真实发送记录；补发会作为历史记录保留 |
| 今日缺口 | `总应发送 - 今日已发送`；当前总应发送未确认，所以不计算缺口 |
| 为什么不再单列额外未发送字段 | 未发送和阻断已经合并进“今日已发送 / 总应发送”以及本节缺口说明，避免重复字段 |

## 内容账本指标怎么读

| 指标 | 定义 | 不能误读成什么 |
|---|---|---|
| GitHub 内容账本累计 | GitHub 中累计登记过的内容条目总数，包括候选、已讲解、已预览、仍排队等状态 | 不能解释成累计发送邮件数量 |
| 已讲解 / 已生成报告与邮件预览 | 已经生成讲解报告，或已经生成邮件预览的内容条目数量 | 不能解释成这些内容都已经进入用户邮箱 |
| 仍在内容队列 | 已进入内容池，但还没有生成讲解报告或邮件预览的条目 | 不能解释成发送失败 |
