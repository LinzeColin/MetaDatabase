## 当前关键结论

| 事项 | 结论 | 对用户的意义 |
|---|---|---|
| Stage 1 arXiv | 已维持验收 | arXiv 单源学习邮件基础仍成立 |
| 本机运行策略 | 本机加本地 Codex 运行器是当前生产策略 | GitHub 不是每日生产运行器 |
| 邮件发送模板 | Email V1 学习邮件模板已进入主分支，M1 到 M4 后续应沿用同一合同 | 用户中心展示 [界面预览](./邮件模板预览.md)；代码链接只作为审查证据 |
| Stage 2 | 尚未正式生产通过 | 不能宣称多来源正式生产 |
| 用户主入口 | GitHub 浅层用户中心 | 不应要求用户去本机目录看状态 |
| 截至今日总候选池 | [299 条总候选记录](./截至今日候选池.md) 已公开；候选队列前20精选已逐篇显示评分 | 用户可以直接在 GitHub 核对全部候选、精选候选、库存流转规则、状态、分数和证据 |
| 各板块数据源 | [数据源与板块健康](./数据源与板块健康.md) 已公开 5 个板块和 6 个数据源 | 当前只有 `SRC-ARXIV` 是生产启用来源；bioRxiv/medRxiv 是影子测试，其他来源是规划 |
| 当前最大剩余缺口 | S2PLT02 真实 proof capture 授权、第二个真实发送日、8 封真实邮件、真实 scheduler proof、S2PLT03 终态 proof、S2PLT04 和最终验收包仍未关闭 | 下一步应先处理 S2PLT02 授权阻断，不把 dry-run、本机证据或 owner packet 误读成生产通过 |

## 用户需要做的决定

| 决策 | 默认建议 | 原因 |
|---|---|---|
| 是否接受当前邮件发送状态模板 | 接受 | 已改为长期口径：今日已发送/总应发送、历史发送记录、模板预览链接、候选池和持续更新规则 |
| 是否继续要求 M1 到 M4 全部使用新模板 | 继续要求 | 这是邮件产品一致性的核心 |
| 是否接受“用户中心是 GitHub 浅层目录” | 接受 | 符合用户不看本机文件的工作方式 |
| 是否现在宣称 Stage 2 生产通过 | 不接受 | 最终门和历史阻断项未关闭 |
| 是否接受当前总候选池公开口径 | 接受 | 299 是截至今日总候选池，前20精选只是按评分抽取的阅读入口，不再混写为总量 |
| 是否接受当前数据源公开口径 | 接受 | 用户中心已经能直接看到每个板块来源、启用状态和证据，不再只依赖深层治理文件 |
