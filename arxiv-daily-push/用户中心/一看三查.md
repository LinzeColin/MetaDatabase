# 一看三查

更新时间：2026-06-26 18:27:28 Australia/Sydney

本页只回答一个问题：今天打开 ADP 后，先看什么、再查什么、哪些结论不能越界。

## 一看：今天能不能继续

| 判断项 | 当前结论 | 含义 |
|---|---|---|
| 今日已发送 / 总应发送 | 2 / 待确认 | 总应发送必须来自当天正式发送计划 |
| 邮件发送模板 | [Email V1 学习邮件模板 UI/UX 预览](./邮件模板预览.md) | 先看用户真实会看到的版式，再看审查证据 |
| 截至今日候选池 | 299 条内容记录；当前展示 11 条 | 候选池包含已生成报告、邮件预览和仍待处理内容 |
| 复习行动收益 | [复习行动与收益](./复习行动与收益.md) 已有 GitHub 展示位、证据链和今日快照待写入字段 | 真实数量必须由当日运行快照写入，不能用测试样例或聊天结论代替 |
| Stage 2 最终门 | 未通过 | 不能宣称正式生产验收完成 |

## 三查：先查这三个页面

| 顺序 | 查什么 | 页面 | 看完应得到什么 |
|---:|---|---|---|
| 1 | 邮件和队列 | [邮件发送与队列状态](./邮件发送与队列状态.md) | 今日已发送 / 总应发送、历史发送记录、模板链接和候选池 |
| 2 | 复习、行动、收益 | [复习行动与收益](./复习行动与收益.md) | 今日快照字段、证据链、已完成能力和仍需真实运行写入的数字 |
| 3 | 路线图和停止门 | [路线图与停止门](./路线图与停止门.md) | 哪些任务可继续，哪些生产结论仍被禁止 |

## 今天可以做什么

| 动作 | 是否可以 | 原因 |
|---|---|---|
| 阅读已发送邮件 | 可以 | 当前已有真实发送记录 |
| 查看今日候选池 | 可以 | 当前已知 299 条内容记录，页面展示 11 条高价值候选 |
| 查看复习、行动、资产、收益字段 | 可以 | 字段和证据链已经放到 GitHub 用户中心 |
| 要求当日运行快照写入真实数量 | 可以 | 当前仓库没有可作为今日真实数字的持久化日报 |
| 宣称 Stage 2 正式生产通过 | 不可以 | 最终门、独立复审和历史阻断项未关闭 |
| 把本机补发成功当成每日生产验收 | 不可以 | 发信成功不等于系统级生产验收 |

## 容易误判的地方

| 看到的事实 | 正确解释 | 错误解释 |
|---|---|---|
| 今日已发送 / 总应发送有进度 | 邮件发送有真实记录，但总应发送仍待正式计划确认 | 整个 Stage 2 已经生产通过 |
| 候选池大于当前展示列表 | 当前页面只展示部分高价值候选 | 系统只剩 11 条候选 |
| 本机计划任务存在 | 本机运行方案有证据 | GitHub 云端已成为每日生产运行器 |
| 复习行动收益有本地证据 | GitHub 已展示字段和证据链，真实数量待当日运行快照写入 | 可以人工猜数字、拿测试样例当今天结果，或宣称正式每日运行已验收 |
