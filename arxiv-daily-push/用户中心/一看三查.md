# 一看三查

更新时间：2026-06-27 22:40:05 Australia/Sydney

本页只回答一个问题：今天打开 ADP 后，先看什么、再查什么、哪些结论不能越界。

## 一看：今天能不能继续

| 判断项 | 当前结论 | 含义 |
|---|---|---|
| 今日已发送 / 总应发送 | 2 / 4 | 总应发送来自 Email V1 每日 3+1 计划目标，不代表 Stage 2 生产验收通过 |
| 邮件发送模板 | [Email V1 学习邮件模板界面预览](./邮件模板预览.md) | 先看用户真实会看到的版式，再看审查证据 |
| 截至今日总候选池 | [299 条总候选记录](./截至今日候选池.md)；候选队列前20精选已列分数 | 总候选池包含已生成报告 / 邮件预览 30 条和待生成 / 待处理候选 269 条 |
| 各板块数据源 | [数据源与板块健康](./数据源与板块健康.md) 已公开 5 个板块和 6 个数据源 | 当前只有 `SRC-ARXIV` 是生产启用来源，影子测试和规划来源不能写成生产启用 |
| 复习行动收益 | [复习行动与收益](./复习行动与收益.md) 已有 GitHub 展示位、证据链和今日快照待写入字段 | 真实数量必须由当日运行快照写入，不能用测试样例或聊天结论代替 |
| Stage 2 最终门 | 未通过 | 不能宣称正式生产验收完成 |

| 查看今日候选池 | 可以 | [截至今日候选池](./截至今日候选池.md) 公开 299 条总候选记录、候选队列前20精选和每日库存流转规则 |

## 容易误判的地方

| 看到的事实 | 正确解释 | 错误解释 |
|---|---|---|
| 今日已发送 / 总应发送有进度 | 邮件发送有真实记录；总应发送为 M1-M4 每日 3+1 计划目标 | 整个 Stage 2 已经生产通过 |
| 总候选池大于前20精选 | 总候选池 299 条；候选队列前20精选只是按评分抽取的阅读入口 | 系统只剩 20 条候选，或 20 条就是总候选队列 |
| 数据源清单显示 6 个来源 | 当前只有 arXiv 来源生产启用；其他来源是影子测试或规划 | Stage 2 多来源已经全部生产通过 |
| 本机计划任务存在 | 本机运行方案有证据 | GitHub 云端已成为每日生产运行器 |
| 复习行动收益有本地证据 | GitHub 已展示字段和证据链，真实数量待当日运行快照写入 | 可以人工猜数字、拿测试样例当今天结果，或宣称正式每日运行已验收 |

计划来源：Email V1 每日 3+1（M1, M2, M3, M4），总应发送 4 封；这不是 Stage 2 生产验收通过声明。
