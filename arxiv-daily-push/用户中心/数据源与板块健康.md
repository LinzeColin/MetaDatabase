做 matched control/candidate、候选至少两次 `HTTP 2xx + parsed_count>0` 而 control 同时仍超时，
才另开 Run Contract 评估最小变更。本轮每个诊断只读请求为 1 个 subrequest、零写入、零新增服务、零付费。

## S3/DAILY_OPERATION 边界

| 边界项 | 当前规则 |
|---|---|
| 来源/板块变更 | 数据源或板块新增、启用、停用只改变来源目录和用户中心展示，不创建或替代持久 DAILY_OPERATION 授权 |
| 持久授权 | 缺 `FINAL_ACCEPTANCE_BUNDLE/daily_operation_persistent_enablement_authorization.json` 时，来源扩展完成也必须保持 `daily_operation_enabled=false` |
| Fail-closed 复核 | 来源/板块变更后仍必须运行 enablement preflight，并确认 `status=FAIL / exit 2` |
| 禁止误读 | 不得把 Stage 2 影子来源、规划来源或来源目录同步写成 S3/DAILY_OPERATION 已进入 |
| v1.2 候选 | `candidate_not_live` 只表示代码与夹具可验，不表示 Google News 已进入 live、已通过 edge canary 或已获部署授权 |
| v1.2 stats-gov 诊断 | `degraded_preserved` / `NO_ADAPTER_FIX` 表示保留失败时降级行为，且 edge 已在零改动下自行恢复、没有因果证据支持修改 live adapter；不表示永久可达或不可达，也不改变来源启停或 D1 当前 health 行 |
| v1.2 PubMed 候选 | `science-advances-pubmed-candidate` 只返回本地/CI 可验的 item + PMID/DOI provenance；现有 `science-advances` Science.org RSS、Worker、cron 和部署均未改，也不表示 NCBI `tool/email` 已完成运营注册 |

## 状态释义

| 状态 | 用户可读含义 | 能否当作生产通过 |
|---|---|---|
| `active` | 已启用，当前可进入正式 arXiv 单源生产维护口径 | 只能对该来源成立 |
| `stage2_test` | 可做 Stage 2 影子测试、复审、no-send 证据 | 不能 |
| `planned` | 已登记方向和口径，等待后续实现或验收 | 不能 |
| 板块 `enabled=false` | 板块已定义但不进入当前生产输出 | 不能 |

## 禁止误读

不能把影子测试或规划来源写成已生产启用；也不能把“来源已登记”写成“Stage 2 多来源生产验收通过”。

| 看到的事实 | 正确解释 | 禁止写法 |
|---|---|---|
| B1 有 3 个来源 | 只有 `SRC-ARXIV` 已启用；bioRxiv/medRxiv 是影子测试来源 | “B1 三个来源都已生产启用” |
| B2/B3/B4 有来源登记 | 这是规划或后续 shadow 的来源清单 | “顶级期刊、中国政策、美国官方信号已经正式生产发送” |
| B5 启用 | B5 是跨板块总览能力 | “B5 有独立采集源” |
| 来源目录存在 | 来源目录是证据和配置镜像 | “来源目录本身就是生产验收” |
