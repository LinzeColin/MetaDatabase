## 状态释义

| 状态 | 用户可读含义 | 能否当作生产通过 |
|---|---|---|
| `active` | 已启用，当前可进入正式 arXiv 单源生产维护口径 | 只能对该来源成立 |
| `stage2_test` | 可做 Stage 2 影子测试、复审、no-send 证据 | 不能 |
| `planned` | 已登记方向和口径，等待后续实现或验收 | 不能 |
| 板块 `enabled=false` | 板块已定义但不进入当前生产输出 | 不能 |

## 禁止误读

不能把影子测试或规划来源写成已生产启用；也不能把“来源已登记”写成“Stage 2 多来源生产验收通过”。

| 看到的事实 | 正确解释 | 禁止写法 |
|---|---|---|
| B1 有 3 个来源 | 只有 `SRC-ARXIV` 已启用；bioRxiv/medRxiv 是影子测试来源 | “B1 三个来源都已生产启用” |
| B2/B3/B4 有来源登记 | 这是规划或后续 shadow 的来源清单 | “顶级期刊、中国政策、美国官方信号已经正式生产发送” |
| B5 启用 | B5 是跨板块总览能力 | “B5 有独立采集源” |
| 来源目录存在 | 来源目录是证据和配置镜像 | “来源目录本身就是生产验收” |

## 更新规则

| 触发变化 | 必须同步更新 | 验收方式 |
|---|---|---|
| 新增、删除或重命名板块 | 本页、[README.md](./README.md)、[一看三查.md](./一看三查.md)、[模型参数文件](../模型参数文件) | 用户中心测试必须覆盖所有板块 |
| 新增、删除或启用数据源 | 本页、[SOURCE_CATALOG.md](../docs/owner/SOURCE_CATALOG.md)、[功能清单](../功能清单)、[开发记录](../开发记录) | `test_user_center_candidate_pool.py` 必须覆盖所有来源 ID |
| 来源从影子测试变成生产启用 | 本页的总览、状态释义、禁止误读和停止门说明 | 必须有独立复审、运行证据和生产门通过记录 |
| 每日运行链路生成新来源健康结果 | 本页需要显示最新健康状态和证据链接 | 不允许只留在本机 JSON 或深层治理文件里 |

计划边界：本页只公开板块与来源清单，不启用 SMTP、不安装调度、不切换生产状态、不修改公共 Schema。
