## 已完成能力

| 能力 | 已完成什么 | 用户价值 |
|---|---|---|
| 复习计划 | 支持到期、未来 7 天、逾期、已完成等状态 | 让论文学习能被重复唤起 |
| 行动闭环 | 支持 15 分钟、2 小时、7 天、30 天行动窗口 | 避免只读不做 |
| 能力资产 | 支持把内容、行动、资产、收益串起来 | 让学习沉淀为可复用能力 |
| 收益复盘 | 区分预期收益和实际收益 | 避免把假设写成已经实现的收益 |
| 周报月报 | 支持阶段性复盘证据 | 支持定期回看学习质量 |

## 收益口径

| 类型 | 可以怎么写 | 禁止怎么写 |
|---|---|---|
| 预期收益 | 基于假设、置信度、行动窗口给出估计 | 写成确定收益 |
| 实际收益 | 有成本、产出和证据时再记录 | 没有证据就填数字 |
| 能力收益 | 记录可复用方法、模板、判断力提升 | 写成空泛成长总结 |
| 今日数量 | 从当日运行报告同步到 GitHub 用户中心 | 用测试 fixture、旧报告、聊天内容或人工猜数代替 |

## 禁止误读

| 事实 | 正确解释 |
|---|---|
| 本页有字段和证据链 | GitHub 用户中心已经能读到结构化入口 |
| 今日数量仍是“待今日运行快照写入” | 当前仓库缺少今天真实日报，不能伪造数字 |
| 阶段记录和运行清单已存在 | 证明本地能力和验收证据，不等于正式生产每日运行 |
| 测试中存在样例数量 | 只能证明代码逻辑，不是今天真实复习或行动数量 |
