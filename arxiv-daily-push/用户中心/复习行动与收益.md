# 复习行动与收益

更新时间：2026-06-26 17:17:15 Australia/Sydney

本页说明 ADP 是否已经把“读论文”变成“复习、行动、能力资产、收益复盘”的闭环。

## 当前结论

| 领域 | 当前状态 | 用户能不能每天直接看 |
|---|---|---|
| 复习计划 | 本地证据已完成，间隔为 1、3、7、14、30、90 天 | 还不能，GitHub 用户中心未自动显示每日到期数量 |
| 行动建议 | 本地证据已覆盖 15 分钟、2 小时、7 天、30 天行动窗口 | 还不能，GitHub 用户中心未自动显示每日行动数量 |
| 能力资产 | 本地证据可追踪内容如何沉淀为能力资产 | 还不能，GitHub 用户中心未自动显示每日新增资产 |
| 收益复盘 | 本地证据要求区分预期收益和实际收益 | 还不能，GitHub 用户中心未自动显示每日转化数量 |
| 周报和月报 | 本地证据已完成 | 还没有成为用户中心固定中文入口 |

## 已经完成的证据

| 事项 | 已完成什么 | 价值 |
|---|---|---|
| 复习计划 | 能计算到期、未来 7 天、逾期、已完成等状态 | 让论文学习能被重复唤起 |
| 行动闭环 | 能把邮件内容转成短、中、长期行动 | 避免只读不做 |
| 能力资产 | 能把内容、行动、资产、收益串起来 | 让学习沉淀为可复用能力 |
| 周报月报 | 能生成阶段性复盘证据 | 支持定期回看学习质量 |

## 还没有达到的用户体验

| 缺口 | 为什么重要 | 推荐下一步 |
|---|---|---|
| GitHub 首页不能自动显示今天到期复习数量 | 用户不知道今天该复习什么 | 增加每日复习数量快照 |
| GitHub 首页不能自动显示今天行动数量 | 用户不知道今天该做什么 | 增加行动数量快照 |
| GitHub 首页不能自动显示新增能力资产 | 用户看不到学习沉淀 | 增加能力资产数量快照 |
| GitHub 首页不能自动显示收益转化 | 用户无法判断投入产出 | 增加收益复盘数量快照 |
| 周报月报没有浅层入口 | 用户找不到周期复盘 | 增加中文周报、月报入口 |

## 收益口径

| 类型 | 可以怎么写 | 禁止怎么写 |
|---|---|---|
| 预期收益 | 基于假设、置信度、行动窗口给出估计 | 写成确定收益 |
| 实际收益 | 有成本、产出和证据时再记录 | 没有证据就填数字 |
| 能力收益 | 记录可复用方法、模板、判断力提升 | 写成空泛成长总结 |
