## 二、数据从哪到哪

来源端点 → 有界获取与原因码 → 解析/去重 → 内容与证据门 → 现有条目存储 → 六主题中文界面 → 只读探针与验收证据。

## 三、关键参数为什么这么定

当前值和改哪里在 `06_运维手册.md`，这里只讲**为什么是这个值**。

| 参数 | 为什么定这个值 |
|---|---|
| current_product_version | 开发期间保持当前线上身份，不预签 v1.2 |
| target_product_version | 所有门和稳定期完成后的目标版本 |
| probe_interval_minutes | 以低成本及时发现只读端点故障 |
| monthly_availability_min | 免费档优先下的内部长期运行目标 |
| probe_coverage_min | 防止缺测被误写成高可用 |
| pipeline_freshness_hours_max | 每日运行允许有限延迟但禁止静默过期 |
| rto_minutes_max | 限定核心只读旅程恢复时间 |
| rpo_hours_max | 每日数据可重放边界 |
| healthy_days_required | 避免短暂绿色被写成长期稳定 |
