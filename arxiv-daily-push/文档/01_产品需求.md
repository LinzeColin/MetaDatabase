# 产品需求

## 一、产品目标

在保留现有 Cloudflare 产品、六主题和数据边界的前提下，以单任务合同闭合三项来源救援、中文人话内容、移动端与视觉回归、版本及运维门；全部阻断门通过后发布 1.2.0，并以 14 个连续健康日形成长期可运行基线。

## 二、给谁用

| 用户 | 他要什么 |
|---|---|
| 负责人 | 每天能在中文界面判断值得看什么、系统是否健康、是否需要行动 |
| 学习者 | 获得诚实、可追证的人话说明，清楚区分中文解释、推断、未知和英文原文 |
| 运维与开发验收者 | 从需求追到测试、证据、构建和部署，并能在故障时回滚 |

## 三、不做什么

- 不恢复 CodexProject 已删除的旧源
- 不迁移到 OVH 或 Coolify，不重建已删除的 Cloudflare 资源
- 不引入付费接口、自动付费升级、模型权重或新数据库
- 不重写六主题，不扩展鉴权、多租户、市场规模、价格或计费
- 不把历史任务状态、绿色测试或实施者自报当成完成证据
