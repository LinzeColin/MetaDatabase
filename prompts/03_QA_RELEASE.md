# v4.2 QA 与发布门禁

只验证已经实现且映射到本次发布的 P0 范围。任何 required test 跳过、无证据或用 fixture 冒充生产数据，均视为失败。

## 必测矩阵

1. **核心路径**：有/空 Watchlist、默认公司、搜索、商业版图、全链供应链、三次递归切换、路径返回、保存/恢复视图。
2. **模型治理**：公式可见、权重和阈值范围校验、影响预览、版本保存、操作日志、原子激活、全局刷新、失败回滚、14 天校准不自动激活。
3. **数据库与语义**：实体/关系/快照/证据/来源/血缘可追溯；方向、有效期、金额、币种、期间、unknown、reported/derived/disputed/revoked 不丢失。
4. **治理工作台**：17 个功能、模型/公式/参数/阈值、关系/供应链/行业/公司范围、任务状态、已解决/未解决、风险、验收和 Gate 可定位。
5. **视口与视觉**：1440x900、1280x800、1024x768；首页视觉覆盖 >=90%，核心系统平均 >=80%；无裁切、遮挡、空白主图或重复导航。
6. **交互反馈**：选择、切换主体、抽屉、保存、模型重算、刷新和回滚的动画连续；loading/success/error 明确；reduced motion 可用。
7. **可访问性**：键盘路径、可见焦点、目标尺寸、非颜色唯一编码、图谱等价列表、对比度和屏幕阅读标签。
8. **性能与稳定性**：定义数据量、设备、浏览器和测量方法；验证图预算、首屏、交互响应、重算进度、断网、超时、部分失败和重复事件。
9. **治理一致性**：运行静态目录校验、JSON/YAML/SQL/API 合同检查、截图/测试证据检查和 checksum 验证。
10. **干净复现**：从 ZIP 解压到新目录，按 README 启动并重跑全部发布门禁。

## 最终输出

- 发布结论：PASS / PASS WITH DEBT / BLOCKED / FAIL。
- 已通过与未通过 Acceptance ID。
- 测试命令、环境、数据规模和证据路径。
- 已解决/未解决、已知风险、控制、停止条件和回滚。
- 更新后的 `VALIDATION_REPORT.md`、`DEVELOPMENT_STATUS.md`、追踪 CSV 和 release note。
