# Workflow

## Implementation Flow

实现。

Implement.

测试。

Test.

审查。

Review.

文档。

Document.

风险说明。

Explain risks.

回滚建议。

Provide rollback advice.

## Required Output

每轮开发结束时必须输出 diff 摘要。

Each development round must end with a diff summary.

每轮开发结束时必须输出下一步建议。

Each development round must end with suggested next steps.

如果测试无法运行，必须说明原因并给出手动命令。

If tests cannot run, explain why and provide manual commands.
