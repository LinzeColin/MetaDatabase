# PFI v0.2.1.1 Stage 6 项目级复审验收

更新时间：2026-06-29

## 定位

用户本轮要求完成 Stage 5 和 Stage 6。当前机器路线锁仍是 `S0` 到 `S5` 六个执行 Stage，其中 `S5` 已吸收真实图表和最终验收。这里的 `Stage 6 项目级复审验收` 是用户口径的第二阶段 closeout：在 Stage 5 完成后，对整体项目做跨板块复审、修复暴露问题，并完成 GitHub、本机 app 和缓存清理。

## 跨板块复审

复审范围：

- 首页总览：净资产、现金余额、投资市值、本月支出、待复核交易、数据源状态。
- 账户与资产：账户入口、资产趋势、空状态和图表数据源。
- 账本流水：筛选、分类、复核、导出和真实流水入口。
- 投资管理：持仓编辑、SQLite 保存、投资趋势和报告同步。
- 消费管理：真实支付宝流水、消费趋势、分类/预算/订阅/异常/现金流入口。
- 数据源与上传：上传中心、导入中心、数据源管理、待复核、导入历史。
- 建议与复盘：建议列表、详情、决策和复盘记录。
- 报告与洞察：月报、季报、年报、自定义报告、导出和持仓报告。
- 市场与研究：市场、研究、策略实验室、单标的回测、参数扫描、盘感训练。
- 设置：数据与系统、隐私、本地存储、反馈、主题、语言和备份恢复。

## Closeout 必须完成

- GitHub main 同步。
- 刷新本机 PFI.app 入口。
- 清理非必要缓存和临时测试库。
- 真实 `PFI.app / 8501` 桌面与移动浏览器验收。
- 正式 operational DB 不写入临时验证持仓。
- README、HANDOFF、开发记录、功能清单、模型参数文件、CHANGELOG 保持一致。

## 本轮复审结果

- Stage 5 合同测试、v0.2.1.1 Stage 0-5 合同回归、用户反馈相关 UI/真实数据回归和完整 PFI 测试均通过。
- 正式 Web Shell 已隔离旧 `合成端到端`、`fixture`、硬编码图表数组和“合成券商/合成导入账本”运行页面文案。
- 真实浏览器矩阵覆盖桌面与移动端，验证一级入口、二级入口、首页功能按钮、全局搜索、策略实验室 canonical route、持仓临时 SQLite 保存后新开页面读回、图表来源和设置隔离。
- 本轮浏览器持仓写入只使用 `/tmp` 临时 SQLite，未写正式 operational DB，临时库已删除。

## 停止条件

以下任一出现，本 closeout 不通过：

- 跨板块复审发现正式页面依赖硬编码图表、假数据或合成验收数据。
- 一级入口、二级入口、主要按钮、搜索、上传、持仓、报告或设置不可用。
- 正式页面出现测试数据、开发者词或旧 AI 演示壳污染。
- GitHub main 未同步，或本机 app 入口未刷新。
