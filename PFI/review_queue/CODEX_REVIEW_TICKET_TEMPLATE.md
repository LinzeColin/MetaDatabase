# Stage 8 - 本地运行 Diff 与 Impacted Metrics / Codex Review Ticket

适用任务：`S8-P1-T1`、`S8-P1-T2`、`S8-P1-T3`、`S8-P2-T1`、`S8-P2-T2`、`S8-P2-T3`、`S8-P2-T4`、`S8-P3-T1`、`S8-P3-T2`、`S8-P3-T3`。

本模板只用于本地复核。无 diff 不联网、不生成 Codex ticket、不触发 LLM。

## 触发原因

填写：业务语义变化、公式逻辑变化、分类冲突、标签冲突、跨板块不一致、测试无法解释之一。

## 影响指标

- P0 核心指标：净资产、生活现金、投资资产、消费总流出、生活消费、投资收益、现金流窗口、待复核数量、Interconnection 异常数量。
- P1 分析指标：分类占比、标签视图、订阅、夜间、大额、商户集中度、投资风格、交易频率、费用拖累、现金拖累。
- P2 展示指标：图表排序、趋势图、辅助说明、tooltip、参数中心展示。
- 不应受影响指标：必须明确列出。例如仅标签显示名变化不应改变净资产、投资收益、现金流窗口。

## 涉及文件

填写变更涉及的 PFI 文件，例如：

```text
PFI/config/pfi_parameters.yaml
PFI/src/pfi_v02/stage_v022_runtime_diff.py
PFI/src/pfi_v02/stage_v022_interconnection.py
PFI/docs/pfi_v02/LEDGER_CLASSIFICATION_STANDARD.md
```

## 期望检查

1. 检查 diff 是否真的改变业务口径或金融数值。
2. 检查 P0 核心指标、P1 分析指标、P2 展示指标是否被混用。
3. 检查首页、消费、投资、现金流、报告是否使用同一事实层。
4. 检查测试失败是否来自业务口径冲突，而不是普通代码错误。

## 禁止事项

- 不得联网。
- 不得调用外部 LLM。
- 不得全仓无差别扫描。
- 不得扩大到 Stage 9-13 功能。
- 不得修改真实交易、支付、券商下单或任何实盘执行路径。

## 中文业务解释

这张票据用于让 Codex 在本地复核重要 diff。目标是确认哪些指标应该受影响，哪些指标不应受影响，避免标签、文案、缓存或图表变化被误报为净资产、投资收益、现金流窗口等核心金额变化。
