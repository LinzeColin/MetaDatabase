# 接近“全网覆盖”需要的条件

严格意义上的“真正无限制全网”无法技术承诺：登录、验证码、付费墙、robots 协议、平台反爬、搜索 API 配额、版权和个人信息合规都会形成边界。可交付目标应定义为：覆盖全网公开可访问信息、已授权登录态信息、官方/API 可检索信息，并保留无法访问来源的缺口记录。

## 必需能力

1. 搜索入口
   - 通用搜索 API：Bing、Google Programmable Search、SerpAPI 或同类服务。
   - 中文搜索：百度、搜狗、360、神马等，需要 API、合规接口或浏览器自动化方案。
   - 垂直搜索：政府网站站内搜索、微信公众号搜索、B 站、抖音、快手、微博、知乎、小红书、今日头条、券商/智库/媒体站内搜索。

2. 平台授权
   - 不在聊天中提供账号密码。
   - 使用 Chrome 已登录会话、平台开放 API、OAuth、或本地受控 cookie/secret 文件。
   - 对 B 站、抖音、快手、微博、知乎、微信公众号、小红书等平台，分别确认是否允许读取公开视频/文章、评论、弹幕、字幕、作者页、搜索结果和互动指标。

3. 合规边界
   - 不绕过付费墙、验证码、访问控制或明确禁止自动化抓取的接口。
   - 尊重 robots、平台条款、速率限制、版权和个人信息保护要求。
   - 对无法访问来源记录 `blocked_by_login`、`blocked_by_captcha`、`paywall`、`robots_disallowed` 等状态。

4. 基础设施
   - 稳定运行环境：常开机器或云服务器。
   - 代理和区域网络策略：仅用于合规访问公开信息，不用于规避封禁或访问控制。
   - 任务队列：分行业、来源、平台、时间、重试优先级。
   - 抓取速率控制：按域名、平台、账号和接口分别限速。
   - 失败重试和断点续跑。
   - HTML/PDF/DOCX/XLSX/OCR/字幕/视频元数据解析。

5. 数据治理
   - 来源权威评分库继续扩容。
   - 每条外部参考保存平台、作者、链接、发布时间、摘录、抓取状态、授权状态、相关度和是否计入报告。
   - 有效参考必须通过强主题匹配；搜索入口、索引页、泛相关政策页、正文太短页面和只命中弱词的页面只能进入线索或缺口队列。
   - 每条未计入报告的外部线索必须进入外部参考缺口队列，标注缺口类型、建议动作、优先级、首次/最近出现运行编号。
   - 查询词优化或 URL 变更产生的新缺口会替代同文件、同来源、同类型的旧 pending 缺口；旧缺口自动标记为 `ignored`，避免 dashboard 保留过期搜索入口。
   - 报告快照保存生成时的来源等级和参考证据，避免后续网页变化影响追溯。

## 当前差距

- 当前来源库仍以中国政府网、国务院信息公开、广东省政府等官方来源为主。
- 中国政府网、人民网、央视网、第一财经等公开站内搜索已接入同域结果解析和公开文章正文抽取；前端脚本型搜索页、验证码、登录和付费墙仍只记录缺口线索。
- B 站公开视频 API 已接入，并会尝试公开视频详情、公开作者页/账号画像、公开字幕、公开评论和公开弹幕摘录；受限评论、完整弹幕历史和其他平台互动数据仍需要授权或平台专用解析器。
- 抖音、快手、微博、知乎、微信公众号、小红书、头条等平台已从单纯搜索入口推进到 `authorized_public_search` 桥接：本地授权 cookie 可用时会尝试读取搜索页、解析公开结果链接、抽取可访问正文，并审计登录/验证码/安全验证/访问控制状态；但各平台专用视频详情、评论、弹幕、作者页、互动指标和稳定正文解析尚未全部形成可计入参考的详情抓取。`config/platform_parsers.json` 和 `reports/platform_parser_dashboard.html` 继续单独追踪视频元数据、字幕、评论、弹幕、作者页、互动数据和正文抽取的 ready/partial/planned 状态。`reports/platform_parser_validation_dashboard.html` 会把 parser 台账与搜索 key、平台授权状态合并，给出下一步是补 key、补授权还是实现解析器；`reports/platform_parser_sample_dashboard.html` 会进一步只读本地样本，判断 parser 是否实际产出可计入报告的高价值证据。
- 附件正文已覆盖 PDF/DOCX/XLSX/PPTX，图片型 PDF/图片附件已接入可选 OCR；`config/attachment_parsers.json` 和 `reports/attachment_parser_dashboard.html` 现在单独追踪 PDF、DOCX、XLSX、OCR、Apache Tika、GROBID 的 ready/partial/planned 状态、业务价值和验收标准。Tika/GROBID 已具备可选 bridge，配置本地服务 URL 后可处理旧版 Office、复杂附件和研究 PDF 引用结构；多平台正文解析仍需继续落地。
- 外部参考缺口已持久化到 `external_reference_gaps`，并进入报告 dashboard、monitor JSON、CLI、独立 HTML 缺口 dashboard、跨运行运营总览 dashboard、平台覆盖矩阵和本地接入验收向导；已支持单条/批量复核、配置 readiness 检查，以及交互式筛选、勾选和命令生成工作台。平台覆盖矩阵会把搜索 API、中文搜索入口、B站、抖音、快手、微博、知乎、微信公众号、小红书、头条和公开站点拆成“已实现能力 / 授权能力 / 阻塞原因 / 下一步动作 / 合规边界”。本地接入验收向导按 P0/P1/P2 展示最少输入、验收命令和业务价值，避免低价值说明挤占页面。后续仍需补平台专用修复动作、真正的后台管理 API 和更细的站点级修复建议。

## 验收口径建议

- 官方来源：中央部委、省级政府、市县政府、重点部门站点全量入库并评分。
- 外部解读：每份报告至少 5 份可计入参考，覆盖至少 2 个外部平台。
- 阻塞透明：无法访问的平台必须进入缺口列表，而不是静默跳过。
- 动作闭环：缺口列表必须能告诉你下一步是补搜索 API key、补本地授权文件、接平台解析器、补附件解析器、优化站内搜索、重试请求还是人工复核 URL。
- 配置验收：补充 key 或授权文件后，必须能用 readiness 命令验证配置存在且不泄露 secret 内容。
- 配置落地：必须能自动生成本地 secret/auth 模板，明确 export 命令、cookie 文件路径和后续验收命令，避免在聊天中传递账号密码。
- 搜索 key 导入：SerpAPI、Bing、Google CSE key 必须能通过 `search-secret-import` 从本地文件或环境变量写入私有 JSON；Google CSE 必须同时具备 API key 和 CSE ID 才算 ready；命令输出和 dashboard 不展示 key 或完整敏感路径。
- 授权导入：浏览器导出的 cookie 文本必须通过 `platform-auth-import` 写入私有目录、设置 0600 权限并更新 `policy-platform-auth.json`；命令输出和 dashboard 只展示脱敏状态，不展示 cookie、session、账号密码或完整敏感路径。
- 覆盖统计：每次运行输出按行业、来源、平台、成功/失败/需授权分类的覆盖报告。
