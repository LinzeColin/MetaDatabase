# 最终交付前需要确认的信息

为了从当前 MVP 一步推进到可长期交付版本，请一次性提供以下信息。给齐后可以开启新的 pursuing goal，我会按“实现、测试、审查、文档、风险、回滚、启用”的顺序连续推进，中间不因普通实现选择反复暂停。

## 必填

1. **报告收件人**
   - Gmail API 发件账号。
   - 收件人邮箱列表。
   - 是否需要抄送/密送。

2. **报告运行策略**
   - 是否启用当前 automation。
   - 运行时间是否固定为北京时间 08:00 和 20:00。
   - 电脑是否能在对应时段保持开机、联网、Codex App 可运行。
   - 是否允许在自动化输出的队列中选择某份报告提前生产。

3. **分析深度**
   - 是否把默认 `ANALYSIS_MODE` 从 `template` 改为 `codex`。
   - 每次最多深度分析多少份文件。
   - 超出部分是否继续排队补跑。
   - 当前自动化版本属于可运行交付版本：能收集、排队、引用外部资料并生成 PDF；如果要达到成熟完整研究报告标准，需要启用 Codex 深度解读、附件正文解析和人工复核规则。

4. **重点行业清单**
   - 你最关心的行业和主题排序。
   - 需要标红的行业、地区、监管主题。
   - 不关心或希望降权的主题。

5. **来源扩容**
   - 是否提供中国政府网“政府网站基本信息下载”的 CSV/ZIP。
   - 是否纳入官方媒体、智库、蓝皮书来源。
   - 是否先覆盖中央+省级，还是继续扩到市县。

6. **解读资料授权**
   - 是否允许使用你的 Chrome 登录态访问 B 站视频页、字幕、评论和弹幕明细。
   - 是否允许使用你的 Chrome 登录态或开放平台 API 访问抖音、快手、微博、知乎、微信公众号、小红书等平台的公开搜索结果和详情页。
   - 是否允许读取本地 cookie 文件；不要在聊天里发送账号密码。
   - 如果必须使用账号，请先在浏览器里登录，或把 cookie 放到受控本地 secret 文件，再提供文件路径。
   - 是否纳入 UP 主白名单/黑名单、官方账号、财经媒体、行业专家账号。
   - 视频/文章解读是否需要抓取播放量、发布时间、作者、简介、评论/弹幕热度、字幕和正文摘录。

## 建议提供

1. **报告风格**
   - 偏投研、战略、合规、销售线索、还是综合管理层简报。
   - 是否需要“一页高管摘要”。
   - PDF 已作为默认主交付格式；仍需确认是否保留同名 HTML/Markdown 旁路产物。
   - 是否需要目录固定展开、按行业分组、按来源分组或按重要性分组。

2. **成本与额度**
   - Codex 深度解读每天最多可使用多少次。
   - 是否允许 OpenAI API 独立计费。
   - 如果触发额度上限，报告是继续用模板模式还是延迟补跑。

3. **附件处理**
   - 是否必须解析 PDF/DOCX/XLSX 正文。
   - 是否需要 OCR 图片型 PDF。
   - 附件保存多久。

4. **交付标准**
   - 最终验收口径：连续成功运行几次算通过。
   - 邮件发送失败时是否重试。
   - 报告和数据库是否需要备份。

## 当前默认假设

- 报告全中文。
- 优先输出 PDF，报告名称体现唯一分析对象并保持简短；同目录保留 HTML 和 Markdown 旁路产物。
- 每份报告只研究 1 份政策文件。
- 每天两次自动化默认生成 2 份单文件研究报告。
- 待生产报告队列按你确认的 40 个行业序号、时间、中央到地方排序，并在报告和 automation 输出中展示。
- 默认只纳入 `2025-01-01` 之后的政策文件。
- 若前序行业出现新文件，系统先回到前序行业，直到该行业 pending 队列清空后再继续后序行业。
- 当前模板分析用于稳定自动化；重大文件进入 Codex 深度解读队列。
- 每份报告有效外部参考不少于 5 份；单纯搜索入口、需登录/验证码来源不计入达标数。
- 网络公开研究/解读资料会作为分析上下文，但权威性低于官方原文。
- 政策报告最低深度标准包括：执行摘要、来源权威等级、政策属性、适用行业和地区、核心条款拆解、产业链影响、受益公司/标的候选、商业影响、交易观察价值、风险和不确定性、与历史政策/既有行研报告的关系、可进入行研或交易系统的任务队列。
- automation 输出必须说明报告是否满足最低深度标准；不满足时列出缺口，并标记是否需要 Codex 深度解读或人工复核。
- Automation 先保持暂停，等你确认后启用。
