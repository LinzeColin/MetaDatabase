## Stage 1 Acceptance Parameters

- 首页总览必须覆盖净资产、账户地图、投资快照、消费快照、数据健康、今日建议。
- 投资管理必须覆盖 Moomoo、支付宝基金、中国券商、ABC Bullion、QBVS 策略实验室。
- 消费管理必须覆盖支付宝、微信、CBA、银行卡、信用卡、订阅，并确保转账不计普通消费。
- 数据源与同步必须覆盖数据源、凭证、同步、导入、对账、待复核和外部只读接口。
- 建议与复盘必须覆盖证据、动作、状态、复盘和失效条件。
- 报告与洞察必须覆盖月度、投资、消费、数据质量、Context Export 和证据链。

## Core Model Parameters

- Credential scopes are read/import only: READ_PROFILE, READ_STATEMENT, READ_BALANCE, READ_POSITIONS, READ_ORDERS, READ_FILLS, IMPORT_FILE.
- Acquisition modes include CSV, ZIP, Excel, PDF, HTML, statement, read-only API/OpenD, watch folder, browser-assisted read and manual snapshot.
- Ledger event types include CASH, TRANSFER, BUY_ASSET, SELL_ASSET, FUND, FEE, TAX, FX, REFUND and VALUATION.
- Snapshots are point-in-time facts and must not overwrite ledger history.

## Classification Rule Parameters

- Own-account transfers are excluded from ordinary consumption.
- CBA to Moomoo, bank to broker and Alipay to bank are transfer events.
- Alipay fund subscription/redemption are fund investment events.
- ABC Bullion gold/silver buy/sell are bullion asset events.
- Credit-card repayment has a dedupe key and is not counted as new consumption.

## Stage 2 Data Source Parameters

- Required source ids: alipay_daily, alipay_fund, moomoo_au, cn_broker, abc_bullion, cba_bank, wechat_pay, other_connector.
- Non-CSV primary sources: alipay_fund, cn_broker, abc_bullion.
- CBA parser version: cba_csv_v1.
- Alipay parser versions: alipay_bill_csv_v1, alipay_bill_zip_v1.
- Low-confidence review choices: A accept suggested classification, B mark as transfer, C mark as consumption, D keep pending.
- Moomoo read-only reuse refs: `PFI/modules/qbvs_lab/qbvs/datasources.py`, `PFI/modules/qbvs_lab/qbvs/moomoo_batch.py`.
- New connectors use plugin profile and field mapping; they must not rewrite the core ledger.
