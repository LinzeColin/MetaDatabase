### Stage 5 计算说明

`消费总流出金额 = 生活消费 + 投资入金 + 基金申购 + 黄金申购 + 投资买入 + 金融费用 - 退款抵消`。

`生活消费金额 = 普通生活消费 - 退款抵消`。

示例：普通消费 `100 CNY`、投资入金 `1000 CNY`、基金申购 `500 CNY`、黄金申购 `200 CNY`、投资买入 `700 CNY`、金融费用 `30 CNY`、退款 `20 CNY` 时，`消费总流出 = 2510 CNY`，`生活消费 = 80 CNY`。

### Stage 5 Stop Condition

- 投资入金、基金申购、黄金申购或投资买入没有进入消费总流出。
- 投资入金、基金申购、黄金申购或投资买入进入生活消费。
- 首页、消费页、报告只显示一个消费数字。
- 分类超过 `L1 ≤ 12`、每类 `L2 ≤ 5`、总 `L2 ≤ 50`。
- 分类缺少 `future_merge_to` 或 `merge_candidate`。
- 任一 L1 的 `primary_category_per_transaction` 不等于 `1`。
- `future_merge_to` 分组无法压缩到 `10` 类或更少。

| reserve_floor_cny | `max(user_min_reserve_cny, average_fixed_monthly_expense_cny * reserve_months)` | 储备金安全线，同时支持用户最小储备金和固定支出倍数。 |
| reserve_months_default | `3` | 默认按 3 个月固定支出建立安全线。 |
| cashflow_pressure_score | `clamp(0,100,100 - 50*reserve_coverage + 20*fixed_cost_pressure + 15*income_uncertainty + 15*large_spend_pressure)` | 现金流压力分，越高表示压力越大。 |
| investment_squeeze_cny | `max(0, reserve_floor_cny - future_cash_balance_after_investment)` | 投资入金挤压生活现金的金额。 |
| required_cashflow_visualizations | `现金流阶梯图, 现金流瀑布图, 储备金安全带, 未来大额流出时间轴, 消费-投资挤压图, 现金流窗口对比表` | Stage 7 锁定可视化合同；真正参数中心页面属于 Stage 9。 |

### Stage 7 Stop Condition

- 置信度权重不等于 100。
- 复核阈值不是统一 `70`，或出现按 source 分层复核阈值。
- 投资入金、基金申购、黄金申购或投资买入没有进入消费总流出。
- 投资入金、基金申购、黄金申购或投资买入进入生活消费。
- 现金流窗口缺少 `7/21/30/60/90/180/360`。
- 储备金安全线不能同时支持用户自定义最小储备金和固定支出倍数。
- 投资入金挤压生活现金无法解释。

## 参数阈值依据

这些阈值是 V0.2 本地 MVP 的“复核触发阈值”，不是最终财务建议结论。它们的设计原则是先减少误判成本：宁可把不确定记录放进待复核，也不把可疑记录自动写成确定分类。

| 阈值 | 当前值 | 位置 | 依据与边界 |
| --- | --- | --- | --- |
| 低置信度复核线 | `confidence < 0.70` | `build_consumption_classification()` | 低于 0.70 的分类缺少足够确定性，进入 A/B/C/D 复核；0.95/0.96/0.98 只用于明确规则命中的本地 fixture。 |
| 未知/默认分类置信度 | `0.45` 到 `0.55` | `classification_rules.py`, parser 输出 | 未知商户、低信息描述或无法确认方向的记录不能自动归类，只能进入待复核。 |
| 大额消费复核线 | `AUD 500` | `build_consumption_anomalies()` | 作为本地初始异常提示线；仅生成“大额消费”复核，不等同于浪费或错误。 |
| 电子产品夜间冲动复核线 | `AUD 300` 以上且夜间/电子产品特征 | `build_consumption_anomalies()` | 用于提示人工复盘高金额、夜间、购物类组合风险；不自动判断消费不合理。 |
| 夜间消费窗口 | `23:00-04:59` | `_hour()` + anomaly rules | 夜间交易更容易需要人工确认；只进入复核。 |
| 月预算 fixture | `AUD 3600` | `build_consumption_summary()` | 临时展示参数，等待用户真实预算替换；不得作为生产预算结论。 |
