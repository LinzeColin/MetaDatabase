## Stage 4 - Economic Event 与 Interconnection 逻辑

Stage 4 的核心目标是把多来源原始记录归并成真实经济事件，并防止同一笔钱在首页、消费、投资、现金流和报告中重复计算。

| 参数 | 当前值 | 说明 |
| --- | --- | --- |
| economic_event_id | `required` | 同一真实经济事件只能有一个 `economic_event_id`；银行扣款、券商入金、持仓增加等多条来源记录可指向同一个真实事件。 |
| interconnection_group_id | `required` | 同一资金链路进入同一个 `interconnection_group_id`，例如 CBA -> Moomoo、支付宝基金申购、退款、信用卡还款。 |
| single_count_rule | `同一 economic_event_id 只进入核心金额一次` | 可多处展示，但同一核心指标口径只能计算一次。 |
| matrix_file | `docs/pfi_v02/INTERCONNECTION_MATRIX.md` | 中文 Interconnection Matrix，列出每类事件对消费总流出、生活消费、投资、净资产、现金流、报告的影响。 |
| metric_dependency_graph | `source_record -> normalized_transaction -> interconnection_group_id -> economic_event_id -> ledger_event -> core_metrics` | Metric Dependency Graph，说明从原始记录到首页/消费/投资/现金流/报告指标的计算链路。 |
| stop_condition_1 | `投资入金未进入消费总流出` | 若投资入金没有计入消费总流出，本 Stage 不合格。正确口径：计入消费总流出，不计入生活消费。 |
| stop_condition_2 | `基金申购未进入消费总流出` | 若基金申购没有计入消费总流出，本 Stage 不合格。正确口径：计入消费总流出，不计入生活消费。 |
| stop_condition_3 | `投资入金错误进入生活消费` | 若投资入金被当成吃饭、购物等生活消费，本 Stage 不合格。 |
| cross_review_agent_1 | `Agent 1` | 复核消费、投资、现金流口径：投资入金、基金申购、投资买入进入消费总流出但不进入生活消费。 |
| cross_review_agent_2 | `Agent 2` | 复核 source -> transaction -> group -> economic event -> ledger -> metric 链路，不允许重复核心计量。 |

### Stage 5 计算说明

`消费总流出金额 = 生活消费 + 投资入金 + 基金申购 + 黄金申购 + 投资买入 + 金融费用 - 退款抵消`。

`生活消费金额 = 普通生活消费 - 退款抵消`。

示例：普通消费 `100 CNY`、投资入金 `1000 CNY`、基金申购 `500 CNY`、黄金申购 `200 CNY`、投资买入 `700 CNY`、金融费用 `30 CNY`、退款 `20 CNY` 时，`消费总流出 = 2510 CNY`，`生活消费 = 80 CNY`。

### Stage 5 Stop Condition

- 投资入金、基金申购、黄金申购或投资买入没有进入消费总流出。
- 投资入金、基金申购、黄金申购或投资买入进入生活消费。
- 首页、消费页、报告只显示一个消费数字。
- 分类超过 `L1 ≤ 12`、每类 `L2 ≤ 5`、总 `L2 ≤ 50`。
- 分类缺少 `future_merge_to` 或 `merge_candidate`。

### Stage 7 Stop Condition

- 置信度权重不等于 100。
- 复核阈值不是统一 `70`，或出现按 source 分层复核阈值。
- 投资入金、基金申购、黄金申购或投资买入没有进入消费总流出。
- 投资入金、基金申购、黄金申购或投资买入进入生活消费。
- 现金流窗口缺少 `7/21/30/60/90/180/360`。
- 储备金安全线不能同时支持用户自定义最小储备金和固定支出倍数。
- 投资入金挤压生活现金无法解释。

## 参数阈值依据

这些阈值是 V0.2 本地 MVP 的“复核触发阈值”，不是最终财务建议结论。它们的设计原则是先减少误判成本：宁可把不确定记录放进待复核，也不把可疑记录自动写成确定分类。

| 阈值 | 当前值 | 位置 | 依据与边界 |
| --- | --- | --- | --- |
| 低置信度复核线 | `confidence < 0.70` | `build_consumption_classification()` | 低于 0.70 的分类缺少足够确定性，进入 A/B/C/D 复核；0.95/0.96/0.98 只用于明确规则命中的本地 fixture。 |
| 未知/默认分类置信度 | `0.45` 到 `0.55` | `classification_rules.py`, parser 输出 | 未知商户、低信息描述或无法确认方向的记录不能自动归类，只能进入待复核。 |
| 大额消费复核线 | `AUD 500` | `build_consumption_anomalies()` | 作为本地初始异常提示线；仅生成“大额消费”复核，不等同于浪费或错误。 |
| 电子产品夜间冲动复核线 | `AUD 300` 以上且夜间/电子产品特征 | `build_consumption_anomalies()` | 用于提示人工复盘高金额、夜间、购物类组合风险；不自动判断消费不合理。 |
| 夜间消费窗口 | `23:00-04:59` | `_hour()` + anomaly rules | 夜间交易更容易需要人工确认；只进入复核。 |
| 月预算 fixture | `AUD 3600` | `build_consumption_summary()` | 临时展示参数，等待用户真实预算替换；不得作为生产预算结论。 |
| 生活现金 reserve floor fixture | `AUD 5000` | `build_cashflow_forecast()` | 临时保守现金底线，防止可投资现金吞掉生活现金；等待真实现金策略替换。 |
| 现金流预测窗口 | `30/90/180` 天 | `build_cashflow_forecast()` | 分别表示短期、中期、半年压力视图；仅用于本地 MVP 展示。 |
