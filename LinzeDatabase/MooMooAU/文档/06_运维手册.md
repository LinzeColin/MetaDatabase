# 运维手册

怎么把它跑起来、参数在哪调、报错了怎么办、都改过些什么。

## 一、怎么跑

改完机器平面，按顺序跑这三条，让人类平面刷新并过关：

```bash
python3 machine/tools/render_human.py            # 先渲染人类平面
python3 machine/tools/check_doc_budget.py        # 体积、中文、纯净三道门
python3 machine/tools/check_blocker_stop.py      # 阻塞重审门
```

## 二、参数在哪调

改一个参数要动哪个文件，都在这。为什么定这个值在 `02_系统架构.md`。

| 参数 | 当前值 | 改哪里 |
|---|---|---|
| 悉尼运行时间 | 悉尼时区每日 04:30 | 机器事实中的运行时间契约 |
| 私有仓数量 | 1 | 机器事实中的仓库契约 |
| 最新图资产上限 | 1 | 机器契约中的时间线发布协议 |
| 生产功能开关 | 全部关闭 | 发布契约中的功能开关 |

## 三、报错了怎么办

| 遇到什么 | 为什么 | 怎么解决 |
|---|---|---|
| 四点半后仍无新鲜证据 | 托管调度排队延迟、事件丢弃或工作流停用 | 保留水位并由下一次运行幂等补偿；周日执行全量对账，异常保持可见 |
| 禁止 Gmail 端点被请求 | 端点守卫或调用边界漂移 | 停止生产、撤销凭证、修复后重跑安全验收 |
| 远端恢复摘要不一致 | 密文或密钥链异常 | 保留 Gmail 原件、禁止 M3、从标准原始数据重建 |
| 时间线固定资产为零 | 串行替换在删除后失败 | 从同一处理后数据快照确定性重建并验证 |
