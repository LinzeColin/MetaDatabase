## 二、数据从哪到哪

行情 Worker 持续订阅 Moomoo OpenD 行情并打上交易所时间戳 → 多个策略 Agent 并行读取行情与研究证据，产出候选信号 → 组合引擎把多策略信号合成为唯一目标仓位（冲突消解、去重）→ 确定性风控引擎逐条裁定（敞口、单笔、白名单、新鲜度、辖区、授权）→ 通过的意图连同幂等键先落库，再进入唯一执行网关 → 执行网关按官方频控调用 OpenD 下单/改单/撤单（Paper 走 SIMULATE，实盘走 REAL）→ 券商回报与成交推送（可能先于同步返回到达）按 `broker_order_id` 与幂等键安全合并进订单状态机 → 对账器核对本地与券商的资金、持仓、未完成订单，差异即停新单 → 每一步的事件写入留痕表并经事务发件箱推送通知 → 控制台与报告只读消费这些落库事实。影子盘全程并行：同一行情、同一策略、同一风控，但把「若实盘会发出的订单」只写库、绝不提交。


## 三、关键参数为什么这么定

当前值和改哪里在 `06_运维手册.md`，这里只讲**为什么是这个值**。

| 参数 | 为什么定这个值 |
|---|---|
| 总敞口上限 | owner 首期授权的资金边界——持仓市值+未成交买单预留+费用+FX 缓冲的硬上限，超过即拒单 |
| 胖手指保护 | owner 已取消固定单笔上限（2026-07-17）；此保险丝不影响任何正常交易（S1 最大自然单笔 1500 AUD），只防程序缺陷把全部资金押进一笔错误订单 |
| 最大持仓数量 | owner 明确不设数量上限；由敞口、单笔、流动性与最小交易单位自然约束 |
| 授权基准币种 | 所有额度按 AUD 折算裁定；跨币种订单按带时间戳汇率保守折算 |
| 默认运行模式 | 失败关闭：仓库默认配置永远不能直接实盘；实盘需 owner 预签授权文件+全部门禁通过后自动激活 |
| 允许市场 | owner 2026-07-17 拍板只做美股首发（摩擦最低、免费行情）；港股待探针核验费用与行情权限后开启；沪深不经 Moomoo AU 路由 |
| 允许订单类型 | owner 决定不限制类型以减少交易摩擦；默认仍用限价控制滑点，市价单保留给止损这类必须成交的场景 |
| 业务频控 | owner 上限：宁缺毋滥、确保出手质量；它是天花板不是目标，策略正常运行远低于此 |
| 官方频控 | Moomoo 官方接口频率限制，防止封禁；与业务频控双层生效 |
| 月度回报门槛 | owner 要求的最低回报率：作为策略晋级实盘的资格线——回测月均达标+真实运行行为一致才自动进实盘，不达标零成本调参循环 |
| 组合日亏断路 | 三层断路器把最坏月份的损失锁在授权资金的一位数百分比 |
| 模拟盘时长要求 | 验证系统真实有效运行且策略速度满足回报门槛（owner 定义的用途） |
| 常驻烤机时长 | 上线前验证 7*24 工程可靠性（心跳、重连、恢复、通知） |
| 部署位置 | owner 电脑不保证 7*24 开机有网；Cloudflare 无常驻进程能力已排除；OpenD 官方支持云部署 |
| 通知与遥控 | owner 无 Telegram，指定邮件为渠道；控制页与邮件指令都只达受控网关（查询/停机/授权确认），永远不能下单 |
| 月度策略评审 | owner 要求每月月底更新调整策略组以符合市场变化；网格+验证+留痕三道纪律防止月度调整变成月度过拟合 |
| 行情新鲜度阈值 | 陈旧行情不得用于风控放行；G3 阶段按实测校准 |
| 付费组件预算 | 第一阶段免费优先；累计净收益覆盖成本前不引入任何付费服务 |
| 策略S1动量得分 | 白盒动量轮动：3/6/12 个月收益加权排序取前 2，绝对动量过滤熊市自动退防 |
| 策略S1标的池 | 高流动性、互相低相关的资产类代表；`BIL` 为现金替身 |
| 策略S1波动率调节 | v2 增强：市场剧烈震荡自动减仓——调研证据显示波动率目标对动量类组合显著改善收益风险比并削尖端风险 |
| 策略S2入场 | 白盒超卖反弹：趋势内极端超卖+收盘贴近全天最低（IBS，v2 增强）+波动足以覆盖佣金才出手，胜率导向；RSI+IBS 组合公开回测显示比单用 RSI 更稳 |
| 策略S2出场 | 先到先执行的三出口，杜绝无限持有 |
| 策略资金分配 | 低摩擦底仓为主、胜率卫星为辅；比例随市场环境按月调整，两者合计受总敞口约束 |
| 费用模型 | 回测与晋级判定必须按此模型扣费；取消单笔上限后 S1 典型单笔约 1500 AUD，一买一卖摩擦降至约 0.2%，全组合月摩擦预估 0.3-0.6% |

## 四、数据长什么样

| 数据 | 关键字段 | 主键 |
|---|---|---|
| `OrderIntent`（订单意图） | 幂等键, 标的, 方向, 数量, 限价, 币种, 策略来源, 创建时间 | `intent_id` |
| `RiskDecision`（风控裁定） | 关联意图, 允许/拒绝, 触发规则, 裁定时敞口快照 | `decision_id` |
| `BrokerOrder`（券商订单） | 关联意图, 券商状态, 已成交数量, 均价, 最后回报时间 | `broker_order_id` |
| `Execution`（成交记录） | 关联券商订单, 成交数量, 成交价, 费用, 成交时间 | `execution_id` |
| `PositionSnapshot`（持仓快照） | 标的, 数量, 成本, 市值, 来源=券商, 快照时间 | `snapshot_id` |
| `CapitalAuthorization`（资金授权） | 总敞口上限, 单笔上限, 币种, 市场白名单, policy hash, 生效与失效时间 | `authorization_id` |
| `JurisdictionCapability`（辖区能力记录） | 账户主体, 所在地, 接口可用性, 买入权限, 证据来源, 判定结果 | `capability_id` |
| `ShadowOrder`（影子订单） | 关联意图, 假想限价, 假想费用, 实盘可行性差异标记 | `shadow_id` |
| `OutboxEvent`（发件箱事件） | 事件类型, 载荷, 投递状态, 投递时间 | `event_id` |
| `ReconciliationRun`（对账批次） | 触发原因, 差异数, 处理结果, 完成时间 | `recon_id` |
