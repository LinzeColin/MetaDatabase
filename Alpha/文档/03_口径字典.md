## 三、恒定为真的规则

| 规则 | 说明 |
|---|---|
| 默认失败关闭 | 仓库提交的默认配置永远是 DISABLED；任何门禁缺失、报错、超时或状态不明，一律拒绝下单而不是放行 |
| 单一执行写者 | 每个实盘账户同一时刻只有一个执行网关实例持有租约；只有租约持有者可以调用券商的下单、改单、撤单接口 |
| 先持久化后提交 | 订单意图与幂等键未落库之前，禁止发起任何券商调用；重启后靠落库记录恢复，杜绝重复提交 |
| 重连先对账 | 断线重连后，账户、资金、持仓、未完成订单全部对账一致之前，不允许提交新订单 |
| LLM 不触钱 | 任何 LLM/研究 Agent 不得持有券商凭据、不得调用下单接口；自然语言输出必须先转成结构化目标仓位，再经确定性风控与执行网关处理 |
| 外部接口不触钱 | 外部/公开 API 与 Dashboard 只能查询状态或在鉴权下请求模式切换与紧急停机，永远不能直接触发真实订单 |
| 秘密不进仓库 | 券商登录密码、交易解锁密码、`acc_id` 敏感记录、token、会话缓存、真实回报原文一律不进 Git |
| 不做监管规避 | 不实现 VPN、代理、地理伪装或任何绕过券商条款的方案；辖区门禁不通过时 Moomoo REAL 买入保持禁用 |
| 真实下单前置门禁 | 环境开关 + 预签授权文件 + policy hash 匹配 + 券商健康 + 辖区能力探针（接口实测）+ 对账一致 + 行情新鲜 + 风控通过 + 幂等键未用 + 紧急停机开关清位 + 执行租约在手，十一项全部通过才可提交 REAL 订单 |
| Paper 与 Shadow 不算完成 | 只有 MICRO_LIVE 真实订单生命周期完整走通，MVP 才算达成；报告、面板、研究成果都不能单独宣布完成 |
| 预签授权自动激活 | owner 预先签发条件授权文件；晋级判定四条全绿时系统自动进入实盘并即时通知 owner，owner 无需再按一次按钮；一键停机随时可推翻自动激活 |
| 白盒纪律 | 所有策略公式、阈值、参数登记在口径字典与配置中，可手工复算；实盘运行中禁止手改参数——改参数 = 新版本号 + 新回测证据 + Git 留痕 + 重新晋级判定 |
| 非法状态即停机 | 订单状态机出现非法迁移、重复语义订单、未知券商订单时，立即触发对账并停止新订单 |
